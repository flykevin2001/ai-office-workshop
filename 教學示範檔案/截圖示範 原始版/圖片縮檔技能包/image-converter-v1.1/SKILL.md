# 圖片轉檔技能 (Image Converter Skill)

這個技能包，是我為了解決自己每天都會遇到的圖片轉檔麻煩而做的。

平常在做教學、寫文章、整理素材時，我很常要截圖、抓網路圖片、整理簡報圖，結果每次都會遇到同一批問題：

- 截圖通常是 `PNG`，檔案很大
- 網路抓下來的圖片常常是一些不太方便直接用的格式，像 `WEBP`
- 有些平台不支援某些格式，貼文章、上傳系統、丟工具時常常卡住
- 有些圖片明明只是要快速用一下，結果一張就 `5MB`、`7MB`
- 有時只是想把圖片壓到 `500KB` 以下，卻還要開好幾個軟體慢慢調

以前我的做法通常是這樣：

- 先用一個工具壓縮 `PNG`
- 如果大小不對，換參數再重跑一次
- 如果平台不吃PNG，還要轉檔

這整個流程最大的問題不是「做不到」，而是「很煩，而且要一直重複」。

像 Adobe Bridge 這類工具當然也能轉，但實際使用時，常常還是要自己管一堆參數。更麻煩的是，不同圖片要用不同判斷：

- 有些圖真的只是太大，壓到 `500KB` 以下就好
- 有些圖雖然已經低於 `500KB`，但因為原本是 `PNG`，其實轉成 `JPG` 還能再小很多
- 有些圖本來就已經是 `JPG`，再跑一次根本沒有意義

最麻煩的就是這種情況：

- 一張 `PNG` 小圖可能已經只有 `400KB`
- 但它實際上轉成 `JPG` 後，可能只剩 `50KB`

如果工具只是單純判斷「低於 `500KB` 就跳過」，這種圖就會被漏掉。結果你還是得自己再多跑幾次、換幾種設定、避免漏網之魚。

所以我最後乾脆把這件事做成一個技能包：讓它自動幫我判斷圖片該不該轉、要不要壓、該用哪種模式，盡量把原本要手動試好幾次的流程，變成一次完成。

此技能提供萬用圖片轉檔流程：將常見圖片格式轉為 `JPG`。它不是單純把圖片一律壓到 `500KB` 以下，而是先判斷「這張圖有沒有值得轉」。

這套技能的核心是預設的 `smart` 模式。它要解決的問題很實際：

- 有些圖片雖然已經低於 `500KB`，但如果原檔是 `PNG`，轉成 `JPG` 可能還能再縮很多
- 有些圖片本來就已經是 `JPG`，而且大小也合格，這種再重跑一次通常沒有意義
- 有些圖片只比門檻大一點，例如 `600KB`，處理目標應該是「安全壓進 500KB」，不是盲目壓到很小

所以這份技能包不是只看檔案大小，而是同時看：

- 原始格式是不是 `JPG`
- 原始檔案有沒有超過大小上限
- 轉成 `JPG` 之後，是否真的有明顯變小

這樣做的好處是：

- 該跳過的會跳過，不浪費時間
- 該轉的會轉，不會錯過 `PNG -> JPG` 這種高壓縮收益
- 批次處理時比較合理，不會把整批圖片都用同一套粗暴規則硬壓

對一般使用者來說，你可以把它理解成一句話：

通常只要你的電腦打得開那張圖片，這個轉檔技能包就能把它轉成 JPG。

## `smart` 模式介紹

`smart` 模式是這份技能包的預設模式，設計給「一批來源格式混雜、大小差異很大」的實際工作情境使用。你不用先人工分類每張圖，它會自動判斷。

### 判斷邏輯

1. 如果原檔已經是 `JPG/JPEG`，而且 `<= 500KB`
   直接跳過，不重跑。

2. 如果原檔不是 `JPG`
   即使目前已經 `<= 500KB`，也會先試著轉成 `JPG` 看看。

3. 轉完之後，如果新 `JPG` 至少比原檔小 `20%`
   就保留新檔。

4. 如果沒有明顯變小
   就視為不值得轉，直接跳過，不保留轉出結果。

### 為什麼不是只看 `500KB`

因為圖片格式本身影響很大，尤其 `PNG` 和 `JPG` 的壓縮特性完全不同。

例如：

- 一張 `600KB` 的 `PNG`，轉成 `JPG` 後可能只剩 `100KB`
- 一張 `400KB` 的 `PNG`，轉成 `JPG` 後也可能只剩 `50KB`

如果只是寫成「低於 `500KB` 就跳過」，這兩張都會被錯誤略過。

反過來說：

- 一張 `480KB` 的 `JPG` 再重新存一次，通常只會有很小差異，甚至還可能變差

這種就應該直接跳過。

### `smart` 模式適合的使用場景

- 你手上是一批圖片，來源格式不一致
- 你希望預設自動做合理判斷，不想每次手動選模式
- 你希望保留品質和效率的平衡，不想把每張圖都暴力重壓
- 你要分享給團隊或社群使用，希望預設行為夠直覺

## 主要模式

### 1. `smart` 模式（預設）

- 如果原檔已經是 `JPG/JPEG`，而且 `<= 500KB`，直接跳過
- 如果原檔不是 `JPG`，即使 `<= 500KB`，也會先試轉一次
- 只有當轉出來的 `JPG` 至少比原檔小 `20%`，才保留轉檔結果
- 適合：
  - `600KB PNG` 可能轉成 `100KB JPG`
  - `400KB PNG` 可能轉成 `50KB JPG`
  - 這兩種都應該轉，不該只因為原檔低於 `500KB` 就跳過

### 2. `limit` 模式

- 標準：`<= 500KB` 直接跳過，不轉、不重存、不改檔
- 適合：一批圖片裡大小差很多，只想處理超標的大圖
- 例子：
  - `200KB` 圖片：跳過
  - `480KB` 圖片：跳過
  - `600KB` 圖片：轉成 JPG，盡量壓到 `500KB` 內
  - `2.2MB PNG`：轉成 JPG，壓到 `500KB` 內

### 3. `aggressive` 模式

- 邏輯：如果檔案只是稍微超過 `500KB`，仍以 `500KB` 為目標
- 但如果是很大的原始檔，預設會參考「原始大小的 `20%`」來決定更小的目標
- 只有當原始檔大小 >= `max-kb * 2.5` 時，這個 `20%` 規則才會啟動
- 例子（預設 `max-kb=500`）：
  - `600KB`：目標仍是 `500KB`
  - `2.2MB PNG`：目標改成 `min(500KB, 2.2MB 的 20%)`，也就是大約 `440KB`

這樣就不會把 `600KB` 的圖硬壓成 `120KB`，但對很大的 `PNG` 又能縮得更有效率。

### 4. `force-jpg` 模式

- 不管原圖是否已低於 `500KB`，都強制輸出 `.jpg`
- 適合：你明明就是要整批統一成 JPG

## 預設行為

- 輸入：常見圖片格式，例如 `PNG`、`JPEG/JPG`、`WEBP`、`BMP`、`TIFF`、`GIF`、`HEIC/HEIF`、`AVIF`
- 輸出：同名 `.jpg`
- 預設大小限制：`500KB`
- 預設保留標準：非 `JPG` 檔案轉出後，至少要比原檔小 `20%`
- 預設策略：
  - 先修正 EXIF 方向
  - 轉成 JPEG 相容的 `RGB`
  - 先嘗試只調整 JPEG 品質
  - 若仍超過限制，再自動等比縮圖直到符合大小

## 單檔使用方式

```bash
# 預設：smart 模式
python3 "scripts/2026-03-11-1627 convert_image.py" "image.png"

# 只看 500KB 門檻，低於就跳過
python3 ".../convert_image.py" "image.png" --mode limit

# 強制全部轉 JPG
python3 ".../convert_image.py" "image.png" --mode force-jpg

# 激進壓縮模式
python3 ".../convert_image.py" "image.png" --mode aggressive
```

## 批次使用方式

```bash
# 處理資料夾內所有支援格式圖片
python3 ".../convert_image.py" "/path/to/images"

# 遞迴處理子資料夾
python3 ".../convert_image.py" "/path/to/images" --recursive
```

## 縮圖行為：預設不覆蓋原檔

使用 `--width`、`--height`、`--max-pixels` 縮圖時，預設會產生一份新的小檔案，檔名加上 `_small` 後綴，原檔保留不動。

例如：`photo.jpg` 縮圖後 → `photo_small.jpg`，原始的 `photo.jpg` 還在。

如果想直接覆蓋原檔，加 `--overwrite`。注意：覆蓋就覆蓋了，原始檔案會不見，無法復原。

```bash
# 預設：產生新檔 photo_small.jpg，原檔不動
python3 ".../convert_image.py" "photo.jpg" --width 800

# 覆蓋原檔（原始檔案會被取代，不可復原）
python3 ".../convert_image.py" "photo.jpg" --width 800 --overwrite
```

注意：如果沒有用到縮圖參數（`--width`、`--height`、`--max-pixels`），輸出行為跟以前一樣：轉檔結果直接存成同名的 `.jpg`。

## 常用參數

```bash
# 改大小上限
python3 ".../convert_image.py" "image.png" --max-kb 300

# 指定輸出寬度（等比），預設產生 image_small.jpg
python3 ".../convert_image.py" "image.png" --width 1600

# 指定輸出高度（等比），預設產生 image_small.jpg
python3 ".../convert_image.py" "image.png" --height 900

# 同時指定寬高（強制輸出該解析度）
python3 ".../convert_image.py" "image.png" --width 1600 --height 900

# 限制總像素數
python3 ".../convert_image.py" "image.png" --max-pixels 2073600

# 覆蓋原檔（搭配縮圖參數使用，原始檔案會不見）
python3 ".../convert_image.py" "image.png" --width 1600 --overwrite

# 指定輸出檔名（單檔用）
python3 ".../convert_image.py" "image.png" --output "image_converted.jpg"
```
