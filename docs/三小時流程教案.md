# AI 辦公室 環境設定課｜三小時教案

> 江江教練設計的「AI 辦公室」上手課，三小時讓你從零安裝、跑出第一個工作流、學會問 Mika 自己摸索。
>
> 本檔為**教案總覽**，含時程、教學重點、現場示範流程。學員看完可以知道：今天我們會做什麼、不會做什麼。

---

## 課程整體節奏

- **總時長**：3 小時
- **江江主講**：2 小時
- **Q&A + 練習**：1 小時
- **不叫「裝機課」**：對外、對內都用「**環境設定**」或「**工作流基礎設定**」

**核心理念**：今天不是要把 20 個技能包全部講完。是要讓學員**離開教室後自己能繼續玩**——會問 Mika、會自己調整技能包。

---

## 課前 5 分鐘｜開場 + 下載連結

**動作**：開門見山給下載連結。

**建議方式（三選一）**：

| 方式 | 優點 | 缺點 |
|---|---|---|
| **LINE 群組**（推薦） | 之後課後答疑可用；下載連結、Q&A 都集中 | 要先建 |
| QR Code 投影 | 現場掃即下載 | 課後追不到人 |
| 直接報網址 | 最快 | 學員手敲容易錯 |

**推薦：開 LINE 群「AI 辦公室 0517 同學會」**

- 把雲端硬碟下載連結固定在公告
- 課後三件事丟群：（1）今天用的技能包清單（2）課堂示範檔（3）下次活動通知
- 學員之間也能彼此問問題（不是只能問你）

## 段一：環境設定（45 分鐘）

> 目的：讓所有人筆電能跑起來 + 知道每個元件是幹嘛的。

### 1-1 三個元件的關係（5 分鐘）

不要叫「裝機」，要說：

> **我們今天要設定的不是「軟體」，是「環境」。**
> 環境 = 終端機 + AI 大腦 + 你的工作資料夾。三個拼起來，才是 AI 辦公室。

| 元件 | 角色 | 比喻 |
|---|---|---|
| **Codex / Claude Code** | AI 大腦（指揮者） | 你的助理本人 |
| **終端機** | 助理跟你電腦講話的接口 | 對話視窗 |
| **AI 辦公室資料夾** | 助理的工作環境 | 你的辦公室 |

### 1-2 下載 AI 辦公室（5 分鐘）

LINE 群裡點連結 → 雲端硬碟 → 下載資料夾 → 放桌面或文件夾。

**不解壓縮 zip，直接資料夾。**（先前確認過 zip 不好用）

### 1-3 Codex 安裝（15 分鐘）

依現場狀況走 Mac / Windows 雙線：
- Mac：終端機指令一鍵
- Windows：先確認 Node 版本，再裝

**現場排錯**：裝不起來、報錯、找不到指令 → 當場解。

### 1-4 Claude Code 安裝（10 分鐘）

同步開始第二個 AI 環境。讓學員理解：**不是非要二選一，兩個都裝以後可以互相備援**。

### 1-5 第一次跟 AI 打招呼（10 分鐘）

- 開終端機 → cd 到 AI 辦公室資料夾 → 啟動 Codex / Claude Code
- 對話框輸入：「**你好**」
- Mika 出場
- 觀察學員反應，這是 AI 辦公室的「**第一次點火**」時刻

## 段二：Codex 新手基本操作（15 分鐘）

> 目的：學員不只會打字，會知道幾個關鍵動作。
> **完整教材** → [[2026-05-17-1027 課程紀錄·教學手冊：Codex 新手基礎設定與小技巧｜Codex、設定檔、技能包、記憶、分支、手機連動|Codex 新手教學手冊（含官方查證 + 江江小技巧）]]

### 2-1 必講四件事（10 分鐘）

1. **設定檔位置**：`~/.codex/config.toml`（個人）/ `.codex/config.toml`（專案）— 開給學員看一眼
2. **沙箱與權限**：新手期先設 `approval_policy = "untrusted"`（每步先問）
3. **必背快捷鍵**：`Ctrl+C`（取消）/ `Ctrl+L`（清螢幕）/ `/clear`（清對話）/ `/fork`（開分支）/ `/resume`（回舊對話）
4. **AGENTS.md = 你寫給 AI 的偏好**：講清楚為什麼 AI 辦公室裡有這個檔（跟你的全域 `~/.codex/AGENTS.md` 概念一樣）

### 2-2 選講（看時間 5 分鐘）

- **手機連動**（2026-05-14 才上線的新熱點，可以當開場吸睛）
- **分支機制 `/fork`**：示範一次給學員看

### 2-3 一個關鍵觀念

> **AI 看不到你的螢幕，AI 只看到你給它的檔案跟你打的字。**

→ 這個觀念決定了之後所有的工作流設計。

### 2-4 容易混淆的三個東西（提一下就好）

- `AGENTS.md` = 你寫給它的指令
- `memories/` = 它自己整理的摘要
- `skills/` = 可重用的能力包

## 段三：第一個工作流——縮圖 + 重新命名（20 分鐘）

> 目的：讓學員親手做一次「我交代 → AI 做 → 我驗收」的循環。

### 3-1 場景設定

學員自備幾張手機照片（或現場給示範圖）：
- 檔名是 `IMG_2031.HEIC`、`IMG_2032.HEIC`、`截圖 2026-03-14 下午4.54.24.png`
- 一團亂、看不出內容

### 3-2 跑兩個技能包

| 步驟 | 技能包 | 學員會說的話 |
|---|---|---|
| 1 | image-renamer（截圖改名） | 「幫我把這些截圖改成中文名」 |
| 2 | image-converter（圖片格式） | 「幫我把 HEIC 轉成 JPG」 |

### 3-3 觀察學員第一次「真的會用」的反應

- 第一次看到 AI 不只是回答，而是**真的動手改檔**
- 第一次體驗「**你不用記指令，你只要說人話**」

## 段四：教大家「怎麼問 Mika」（20 分鐘）⭐ 核心段落

> 重點：**不要由我介紹 21 個技能包，要教學員問 Mika，讓 Mika 介紹自己。**
> 這就是 AI 辦公室的本質——你不用記功能，AI 會教你它有什麼能用的。

### 4-1 三句話打開 Mika 的全部能力

讓學員逐句試：

1. 「**這裡有哪些助理可以幫我？**」
   → Mika 列 21 個助理 + 各自用途

2. 「**我手上有一份 [XXX]，可以怎麼處理？**」
   → Mika 推薦對應技能包 + 操作步驟

3. 「**我想[XXX]，有沒有什麼能幫我？**」
   → Mika 找出相關技能包 + 主動建議

### 4-2 給學員一個「**萬用觸發詞**」

學員不知道要說什麼時，記這句：

> **「我要怎麼開始？」**

→ Mika 永遠會回應，永遠會引導下一步。

### 4-3 練習時間（10 分鐘）

讓每個學員選一個自己的場景試問 Mika，互相分享 Mika 回了什麼。

## 段五：自己改造技能包（20 分鐘）

> 目的：教學員「以後不用再來上課，自己就能跟著系統長大」。

### 5-1 三種「自己改技能包」的方式

1. **小改**：直接跟 Mika 說「這個技能包的觸發詞我想加一個 XXX」
2. **中改**：跟 Mika 說「我想把這個技能包改成適合我自己的版本」
3. **大改**：跟技能包大師（skill-master）說「我有個新工作流，幫我做成技能包」

### 5-2 為什麼這段重要

> 這堂課的真正交付，不是 21 個技能包。是「**讓你回家之後，技能包會繼續長**」這件事。

## 段五·5｜GitHub 進階補充（10 分鐘，做完網頁簡報後接著講）

> 順序提醒：**Q&A 裡如果有人做了網頁簡報 demo**，就接著講這段。如果沒人做，等課後 LINE 群再丟連結補。

### 5.5-1 為什麼今天提 GitHub

不是要教學員當工程師，而是讓他們知道：
- **網頁做出來要放哪裡？** → GitHub Pages 是免費好選擇
- **技能包想分享給朋友？** → GitHub 是最方便的傳遞方式
- **想看別人怎麼做？** → 很多技能包、AI 工具都在 GitHub 上公開

### 5.5-2 GitHub 不是專門做網頁的（要先講清楚）

> 「**GitHub 不是專門做網頁的。它最一開始是『備份管理程式碼版本』的工具。**」
>
> 「網頁、技能包、文件，這些都可以放上去——所以**功能比較多**，但缺點就是：**網頁不是最漂亮的、網址也不是最好看的**。」
>
> 「其他做網頁的服務有 Vercel、Netlify 等等，今天先不講。」

### 5.5-3 GitHub 的多種用途（一句話帶過）

| 用途 | 一句話說明 |
|---|---|
| **版本管理**（最原始用途） | 你改了什麼、什麼時候改、改錯了可以救回來 |
| **網頁發布**（GitHub Pages） | 免費、永久、有網址，但版面要自己做 |
| **技能包分發** | 把你的 SKILL.md 放上去，別人下載就能用 |
| **訂閱別人的作品** | 喜歡誰的東西可以關注，他改了你會收到通知 |
| **接案作品集** | 把作品放上去，求職、接案都看得到 |

### 5.5-4 GitHub 註冊好就能用嗎？

- **瀏覽器版**：✅ 註冊好就能用——建 repo、看程式碼、開 Pages 部署網頁
- **讓 Codex / Claude 自動操作**：要再裝一個小工具 `gh`（GitHub CLI）
  - 今天不裝沒關係，回家想用時對 Mika 說「**教我裝 gh**」
  - AI 辦公室有 `github-ops` 技能包，裝完 gh 之後可以一鍵同步

### 5.5-5 加速指令：github-ops 技能包

對 Mika 說：

- 「**幫我把這個資料夾上 GitHub**」 → github-ops 自動跑（建 repo、掃敏感檔、commit、push）
- 「**幫我發布到 GitHub Pages**」 → 自動部署網頁
- 「**幫我同步技能包到 GitHub**」 → 把你的技能包打包公開

### 5.5-6 課堂示範（看時間決定要不要）

如果時間夠：現場給一個 demo——用 github-ops 把剛剛學員做的小東西上 GitHub Pages。
時間不夠：丟 LINE 群「回家自己玩，卡住問 Mika」。

## 段六：Q&A + 練習（60 分鐘）

> 這 1 小時是課程的靈魂。學員真的有問題，比我講什麼都重要。

### 6-1 收集問題清單（10 分鐘）

請每個學員寫下：
- 一個你今天「**最想當場做成**」的事情
- 一個你「**還聽不懂**」的地方
- 一個你「**回家想試但不確定怎麼開始**」的場景

### 6-2 現場分派（5 分鐘）

把學員需求分三類：

| 類型 | 處理方式 |
|---|---|
| **具體場景**（A 想做 B） | 我帶他現場跑一次 |
| **概念不懂** | 我用比喻再講一次 |
| **回家想試但卡住** | 教他怎麼問 Mika 在家自己 debug |

### 6-3 現場手戳示範（35 分鐘）

依現場提問實時選技能包：
- 簡報製作 → spring-editor / huashu-design 概念
- 圖卡生成 → social-carousel-card（教他這個是 Codex 走 image_gen 的路線）
- 個人設定 → personal-profile-builder + learning-map-builder（三塊自我提煉）
- 文件整理 → personal-director + tag-manager

### 6-4 Fallback：如果學員都沒想法（10 分鐘）

退而求其次的三個現成示範：

1. **生圖**：「幫我生一張 AI 辦公室的封面圖」
2. **做簡報**：「把這份 [XXX 文件] 拆成 5 頁的簡報」
3. **寫企劃**：「我要做一個 [XXX] 的小活動，幫我擬企劃」

## 結尾 5 分鐘｜帶走清單 + 金句

學員離開教室前，確認帶走三件事：

1. **LINE 群連結**（課後 7 天答疑、未來活動通知）
2. **下載連結**（AI 辦公室資料夾）
3. **一句萬用咒語**：對 Mika 說「**我要怎麼開始？**」

### ⭐ 結尾金句（江江親說）

> **我們從「問 AI」，轉向「管理 AI」。**
>
> **而最終無可取代的，是整理出我們自己的知識。**

這句話濃縮今天三小時的所有內容：
- 「問 AI」= 你過去用 ChatGPT 的方式
- 「管理 AI」= 今天教的 AI 辦公室、Mika、技能包、工作流
- 「整理出我們自己的知識」= 為什麼要有資料夾結構、檔名規則、3X4、AGENTS.md

呼應招生頁那句：**「你不是要去學 AI，是要讓 AI 來學你。」**
