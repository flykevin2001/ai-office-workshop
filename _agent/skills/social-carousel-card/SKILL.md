# 通用社群輪播卡

用途：把已成形的文章或貼文，拆成可發布的社群輪播卡。預設先規劃、再生成；生成時一張一張出，不合併成拼圖。

## ⚠️ 適用環境（很重要）

這個技能包走的是「**呼叫生成模型直接出圖**」的路線（用 image_gen 等工具）。所以：

| 環境 | 行為 |
|---|---|
| **OpenAI Codex / ChatGPT** | ✅ 直接生圖。這是這個技能包**主要設計給的環境** |
| **Claude Code / Cowork**（本地） | ⚠️ 沒有 image_gen 工具，會幫你**產生圖卡生成 prompt**，你再把 prompt 拿去 ChatGPT / Codex / Midjourney 等生圖工具用 |
| **Claude 網頁版（claude.ai）** | ⚠️ 同上，沒有生圖能力 |

**跟「用 Claude 網頁版做圖卡」不同**：Claude 網頁版做圖卡是用 HTML/CSS 程式碼產出網頁，再透過截圖工具截成圖片（精準度高、版面可控，但工序較長）。這個技能包走的是「直接生成圖片」路線（速度快、視覺豐富，但細節較難精準控制）。

**選哪個？**
- 想要**速度快、視覺有變化** → 用這個技能包（走 Codex / ChatGPT 生圖）
- 想要**版面精準、文字必須完全可控** → 用 HTML 圖卡截圖路線（Claude 環境的另一套流程）

## 必讀

- `references/carousel-planning.md`：張數、平台、受眾與流程選擇
- `references/brand-inputs.md`：品牌資產、語氣、CTA 與可寫回設定
- `references/generation-prompts.md`：逐張生成 prompt 規格

## 預設設定

- 預設比例：直式 4:5，適合 IG 與 Threads 共用。
- 可選比例：正方形 1:1、橫式 16:9 一頁總結、自訂比例如 4:6。
- 預設流程：先列輪播規劃，等使用者確認後再生成。
- 預設文案：使用者已貼上的文案視為素材與主要語意來源，不主動重寫。若文案還沒定稿，先建議使用者定稿後再拆卡。
- 預設輸出：逐張生成。五張就是五張獨立圖片，不做九宮格、拼貼圖或多卡合併。

## 工作模式

1. **規劃模式**
   - 先列張數、每張主題、主標、內文重點、版面類型、視覺元素與 CTA。
   - 不直接生圖，等使用者確認。

2. **第一張試跑模式**
   - 使用者不確定風格時，先生成第 1 張封面卡。
   - 第 1 張確認後，再自動依同風格逐張生成剩餘卡片。

3. **全自動模式**
   - 使用者明確說「全部全自動」「直接出完」時，可以略過逐步確認。
   - 仍要保持逐張生成，不可把多張卡做在同一張圖。

## 核心流程

1. 判斷平台與比例：預設 4:5；若使用者指定 IG、Threads、正方形、一頁總結或自訂比例，照指定。
2. 判斷張數：
   - Threads / 快速社群觀點：3-5 張。
   - IG 教學型輪播：7-10 張。
   - 受眾熟悉、閱讀速度快、資訊密度高：3-5 張。
   - 受眾新手、需要慢慢鋪陳：7-10 張。
3. 先交付輪播規劃表。
4. 使用者確認後，依卡序逐張產圖或逐張產生圖 prompt。
5. 若需要一頁總結，最後再產一張 16:9 橫式總結。

## 完成回報

回報時列出：

- 使用的流程模式
- 已完成的卡片清單
