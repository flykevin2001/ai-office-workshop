## 核心理念

把模糊的問題變精確。

根本原則：逐詞確認，不放過任何一個可能有模糊空間的詞。

- 每一個名詞：這個詞具體指什麼？
- 每一個動詞：這個動作具體怎麼做？誰做？
- 每一個形容詞：這個標準是什麼？誰定義？
- 每一個情緒性表達：背後具體的情況是什麼？

不是套框架，是逐詞確認。

## 適用情境

當有人問你問題，問得很模糊時：
- 「幫我拆解這個問題」
- 「這個人的問題在哪」
- 「他問的太模糊了」
- 「幫我釐清這個需求」

## 核心價值

1. 把模糊變精確：讓問題清楚到可以直接執行
2. 附帶效果：在整理問題的過程中，常常自然就會有答案或靈感
3. 情緒是線索：從情緒性表達追問出具體數字和情境

## 使用方式

### 方式一：自己用
問問題前先拆一遍，把每個詞都確認清楚。

### 方式二：給別人用
別人問你問題時，用逐詞拆解幫他釐清。可以直接把拆解後的問題清單丟給對方。

### 方式三：做成機器人
讓 AI 自動用逐詞拆解的方式提問，幫你過濾掉還沒想清楚的人。

## 與 AI 問題拆解技能包的差異

- problem-decomposer（本技能包）：通用的問題拆解，核心是「逐詞確認」
- ai-problem-decomposer：專門針對「AI 怎麼做 XX」的問題，用四個關卡檢查

兩者可以搭配使用：
1. 先用 problem-decomposer 逐詞拆解，把問題變精確
2. 如果是 AI 相關問題，再用 ai-problem-decomposer 的四個關卡檢查
