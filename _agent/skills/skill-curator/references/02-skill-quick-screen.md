# Skill 快篩細則

## 七問清單

對每個待篩選的新技能包，依序問：

### 一、工作流匹配

1. **這包在使用者主要目標路徑上嗎？**
   - 是 → +1
   - 繞遠路 → -1
   - 完全無關 → 直接 ❌

### 二、現有重疊

2. **跟現有哪些技能包功能重疊？重疊比例多少？**
   - 重疊 < 30% → 不算問題
   - 重疊 30-70% → 🟡 調整後收（合併或限定範圍）
   - 重疊 > 70% → ❌ 不收，使用既有的

### 三、使用者偏好

3. **使用者最怕的坑這包會踩到嗎？**（依 profile 第三題）
   - 會 → ❌
   - 不會 → +1

4. **風格、語氣合不合 profile？**
   - 合 → +1
   - 不合，但可改 → 🟡
   - 不合，難改 → ❌

### 四、品質基本面

5. **SKILL.md 有清楚的觸發條件、執行步驟、輸出格式嗎？**
   - 三項都有 → +1
   - 缺一項 → 🟡（補上後可收）
   - 缺兩項以上 → ❌

6. **references/ 結構合理嗎？**（如果有的話）
   - 三層載入清楚（SKILL.md 決策 / references 操作） → +1
   - 全擠 SKILL.md → 🟡（建議拆）
   - 沒有 references → 看技能包複雜度

7. **觸發詞會跟現有技能包打架嗎？**（接模式一）
   - 不會 → +1
   - 會但好改 → 🟡
   - 會且難改 → ❌

## 決策矩陣

```
分數 ≥ +3，無 ❌    → ✅ 直接收
分數 0 ~ +2，無 ❌ → 🟡 調整後收（列出該改的點）
分數 < 0 或有 ❌    → ❌ 不收（說明拒絕理由）
```

## 個人設定檔的 fallback：快速 mini-profile

沒有完整 profile 時，問三題：

1. **你主要想做什麼？**
   - 一句話描述當前主要工作或產出

2. **想要達到什麼效果、目標？**
   - 半年內想看到的具體成果

3. **最怕踩什麼坑？**
   - 過去踩過的坑 / 看到別人踩過想避開的

用這三題的回答跑這次快篩，跑完不存。

## 快篩報告格式

```
📋 Skill 快篩報告
待篩技能包：[名稱]
來源：[URL 或路徑]
profile：[完整 profile / 快速 mini-profile]

—— 七問評分 ——
1. 工作流匹配：+1（理由）
2. 現有重疊：30%（跟 X 包）
3. 使用者偏坑：不會
...

—— 決定 ——
🟡 調整後收

—— 改的點 ——
1. description 加限定詞「XXX」避免跟 X 包打架
2. references/ 第二層該拆

—— 下一步 ——
轉技能包大師執行修改
或
使用者勾選略過修改直接收
```

## 不收的情境（直接 ❌）

以下任一情況可直接拒絕，不用走完七問：

- description 不知道在做什麼（自己讀完還是看不懂）
- 沒有 SKILL.md，只有一堆 prompts
- 跟現有技能包重疊 > 70%
- 會踩到使用者明示的坑
- 維護者很久沒更新（GitHub 半年沒 commit、issue 堆積無人理）
- 授權不明（公開版要看 LICENSE）
