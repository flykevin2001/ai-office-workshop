# 觸發詞健檢細則

## 衝突判定三層

### Level 1：完全相同（直接衝突）

兩個以上技能包的 description 或觸發詞段落出現一模一樣的字串。

例：
- A 包：「當使用者說『整理』時觸發」
- B 包：「當使用者說『整理』時觸發」

→ 必修。同一個詞不能對應兩個技能包。

### Level 2：近義詞（疑似衝突）

常見近義詞對：

| 詞 A | 詞 B | 處理 |
|------|------|------|
| 整理 | 歸檔 | 限定範圍：「整理對話」vs「歸檔對話」 |
| 寫 | 製作 | 限定產出：「寫文章」vs「製作圖卡」 |
| 檢查 | 健檢 | 限定對象：「檢查 X」vs「健檢 Y」 |
| 找 | 搜 | 限定來源：「找筆記」vs「搜網路」 |
| 做 | 建 | 限定範疇：「做圖」vs「建技能包」 |
| 改 | 更新 | 限定動作：「改標題」vs「更新規則」 |

→ 視情況修，若兩包確實做不同領域則可保留，但 description 要加限定詞。

### Level 3：過多觸發詞（待收斂）

一個技能包超過 **5 個觸發詞** 算過多。建議：

- 主觸發詞 1-2 個（必觸發）
- 替代觸發詞 2-3 個（在 description 末段列出）
- 隱性觸發（情境語句）放在 description 中段，不算數量

## 健檢執行步驟

### Step 1：建立觸發詞索引

```
技能包名稱 | 主觸發詞 | 觸發詞總數 | description 字數
----------|---------|----------|---------------
A          | 整理     | 6        | 180
B          | 歸檔     | 3        | 120
C          | 寫文章   | 8        | 250
```

### Step 2：跑三層檢查

對每個觸發詞，跨技能包比對。建議用簡單 grep：

```bash
grep -rE "當使用者說|觸發詞|description:" _agent/skills/*/SKILL.md
```

### Step 3：產出健檢報告

格式：

```
📋 觸發詞健檢報告
掃描時間：YYYY-MM-DD HH:MM
技能包總數：N

🔴 Level 1 直接衝突（M 組）
🟡 Level 2 疑似衝突（K 組）
🟢 Level 3 觸發詞過多（L 個）

—— 修正建議 ——
針對每組／每個，給出 1-2 個修法。使用者勾選 → 轉技能包大師。
```

## 收斂原則

當需要收斂一個技能包的觸發詞時：

1. **保留最具體的**：「健檢觸發詞」比「健檢」更具體，保留前者
2. **去掉太通用的**：「整理」「處理」「做」這類萬用詞除非搭配限定，否則砍
3. **保留使用者口頭習慣**：使用者實際會怎麼說？日記裡找線索
4. **保留差異性**：跟其他技能包能形成明顯區別的詞優先留
