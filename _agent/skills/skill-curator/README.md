# 技能包大總管（Skill Curator）

[English version](https://github.com/Jiang-Yude/skill-curator-en)

管整個工作流裡所有技能包之間的關係。不創建單一技能包（那是技能包大師的事），只管整個知識庫裡技能包該怎麼共存。

## 這個技能包要解決什麼問題

當你開始用 AI agent 工作流，技能包會從三個地方同時冒：自己寫的、網路抓的、AI 順手生的。久了會出三件麻煩事：

- **觸發詞打架**：兩個技能包都用「整理」當觸發詞，AI 不知道叫哪個
- **重複堆積**：A 包跟 B 包做的事 80% 重疊，但都還留著
- **規格漂移**：實際工作流變了，但技能包 SKILL.md 還是舊的，AI 用了老地圖

這個技能包專門處理這三件事，加上一個閉環：模式四會掃最近工作日記，主動偵測技能包該不該更新。

## 四種模式

| 模式 | 觸發 | 做什麼 |
|------|------|------|
| 一、觸發詞健檢 | 「健檢觸發詞」 | 掃 description，列衝突清單與收斂建議 |
| 二、Skill 快篩 | 「Skill 快篩」「這個技能包要不要收」 | 跟工作流比對，三選一決定（收／調整後收／不收） |
| 三、整併與拆分 | 「整理我的知識庫技能包」 | 找太近的（合）、太雜的（拆）、太久沒用的（封存） |
| 四、更新檢查 | 「檢查技能包要不要更新」 | 掃最近材料，兩道 gate 過濾，列待寫入規則 |

## 重要設計

### 模式二依賴個人設定檔

「這包適不適合我」這個問題，沒有個人設定檔（user-profile）就答不出來。個人設定檔技能包由使用者另外建。沒有時，本包會提供「快速 mini-profile」三題版本當作 fallback。

三題：
1. 你主要想做什麼？
2. 想要達到什麼效果、目標？
3. 最怕踩什麼坑？

### 模式四採有界記憶原則

每次更新都是取捨。新規則要進來，必須回答：

- **值不值得記**：重複出現過嗎？未來會用到嗎？是規則還是當下狀態？
- **要擠掉誰**：哪一段過時可刪？哪一段可濃縮？

沒人能擠掉 → 暫不寫入，進候選池下次再評。寧可少寫，不亂塞。

### 模式四要先問範圍

```
⚠️ 這是大工程，先選範圍：
1. 🎯 特定某篇技能包
2. 📓 最近工作日記
3. 📁 最近常用專案
4. 🌐 整個知識庫（建議每週用量還剩很多、感覺用不完時再做）
```

## 使用方法

1. 下載這個 repo 到你的 agent 環境（Claude Code / Codex / Cursor 等支援 skill 的工具）
2. 放進 skills 目錄（例如 `_agent/skills/skill-curator/`）
3. 觸發任一模式：跟 AI 說「健檢觸發詞」「Skill 快篩」「整理我的技能包」「檢查技能包要不要更新」

## 跟其他技能包的關係

- **skill-master（技能包大師）**：互補。本包列出待動清單，由 skill-master 執行實際修改
- **user-profile（個人設定檔）**：強依賴。模式二必須讀 profile 才能判斷
- **其他所有技能包**：治理對象。本包不修改任何技能包內容，只列建議
