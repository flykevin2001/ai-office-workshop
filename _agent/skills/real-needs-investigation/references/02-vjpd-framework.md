# VJPD 行動框架詳解

VJPD 是訪談與問卷的骨架。三大原則告訴你「怎麼問」，VJPD 告訴你「該問哪四件事」。

---

## VJPD 四個元素

| 元素 | 全名 | 目的 | 一句話定位 |
|---|---|---|---|
| **V** | Validate（驗證問題） | 確認痛點真實存在 | 這個問題真的有發生過嗎？ |
| **J** | Judge（判斷影響） | 量化痛點嚴重性 | 這問題嚴重到什麼程度？ |
| **P** | Probe（探究行為） | 挖掘現有解決方案 | 對方現在怎麼處理？ |
| **D** | Demographics（用戶畫像） | 釐清受訪者背景 | 對方是誰、處在什麼情境？ |

四個元素缺一不可：
- 缺 V：你不知道問題真不真
- 缺 J：你不知道值不值得解決
- 缺 P：你不知道你的方案是不是比對方現有方法更好
- 缺 D：你不知道你訪談的人是不是你的目標客群

---

## V — Validate（驗證問題）

### 目的
確認痛點是真的、不是你腦補出來的。

### 核心問題模板
- 「您上次遇到這個問題是什麼時候？」
- 「最近一次發生是怎樣的情況？」
- 「您可以描述一個具體的場景嗎？」
- 「在處理 OO 的時候，最頭痛的是什麼？具體遇到哪些困難？」

### 紅旗訊號
對方答不出具體事件、只能講「就……一直都會啊」「常常啊」 → 痛點可能不夠尖銳，或對方記不清楚（代表發生頻率低）。

---

## J — Judge（判斷影響）

### 目的
量化痛點代價。讓對方告訴你這個問題「值不值得花錢解決」。

### 核心問題模板
- 「這個問題讓您損失了多少時間／金錢？」
- 「如果這個問題沒解決，對您的工作有什麼影響？會造成什麼損失？」
- 「上次這個問題發生時，後續產生了哪些連鎖反應？」
- 「為了解決它，您願意（或曾經）付出多少？」

### 量化的四個維度
1. **時間**：每次花多久？一週幾次？一年累積多少小時？
2. **金錢**：直接成本？間接損失？機會成本？
3. **情緒**：影響心情、睡眠、人際關係？
4. **後果**：被罵、被退件、客戶流失、專案延誤？

### 紅旗訊號
對方答「還好啦」「也不會怎樣」「就忍一下」 → 痛不夠痛，假需求。

---

## P — Probe（探究行為）

### 目的
摸清對方「現在的解決方案」。
你的產品要贏，得贏過對方現在用的東西，包含「什麼都不做」。

### 核心問題模板
- 「您現在都怎麼處理這些回饋？可以詳細描述一下完整流程嗎？用什麼工具？」
- 「上次遇到時，第一步做了什麼？第二步？第三步？」
- 「過去你曾經嘗試過哪些方法？哪個最有效？哪個最沒效？」
- 「這個方法你最不滿意的地方是什麼？」

### 黃金追問
當對方講到「現在用 XX」時，馬上追問：
- 「最沒效率／最痛苦的是哪個環節？」
- 「為什麼還繼續用這個方法？」
- 「上次有想過換掉它嗎？為什麼最後沒換？」

對方換不掉現有方案的原因，才是你的產品要解決的真正障礙。

---

## D — Demographics（用戶畫像）

### 目的
釐清「誰」在跟你講這些。讓你知道訪談結果是來自目標客群還是路人。

### 核心問題模板
- 「您一週大概有幾堂課？每堂課多少人？」
- 「您在這個領域工作多久了？」
- 「您負責這類事務多久？」
- 「您團隊規模？產業別？年資？」

### 重點
D 的問題通常放在最後問，因為前面有事實鋪墊，對方比較願意分享背景。
但 D 的資訊在分析時，會用來「分群」——同樣的痛點，可能不同畫像的人有不同強度。

## VJPD 訪談問題範本

### 情境：驗證「組織知識管理」需求

```
V - 驗證痛點
- 「可以聊聊上一次有同事離職、帶走重要知識的經驗嗎？」
- 「那次具體發生了什麼事？」

J - 判斷嚴重性
- 「後來花了多少時間重建那些知識？」
- 「如果用金錢衡量，那次損失大概多少？」
- 「除了時間金錢，還有沒有其他層面的影響？（例如客戶信任、團隊士氣）」

P - 探究現況
- 「你們現在怎麼保存重要的工作經驗？」
- 「最近一次同事交接，是怎麼進行的？流程？」
- 「這個方法有什麼不滿意的地方？」

D - 了解背景
- 「你們團隊有多少人？」
- 「你負責這類事務多久了？」
- 「平均一年人員流動率大概多少？」
```

## VJPD 問卷範本（量化驗證階段用）

訪談後要做大樣本驗證時，把訪談發現轉成可量化問題：

### 頻率驗證（V）
```
您每週需要處理 OO 的次數？
□ 1 次以下 □ 1-2 次 □ 3-5 次 □ 6 次以上
```

### 時間成本（J）
```
每次處理 OO 平均花費時間？
□ 30 分鐘內 □ 30-60 分鐘 □ 60-90 分鐘 □ 90 分鐘以上
```

### 痛點確認（J）
```
以下哪些是您處理 OO 時的主要困擾？（可複選）
□ 內容重複太多 □ 格式不統一 □ 難以找到重點
□ 彙整耗時 □ 分析困難 □ 其他___
```

### 現有解決方案（P）
```
目前主要使用的工具？（可複選）
□ Excel □ Word □ Google 表單 □ 截圖整理
□ 手寫筆記 □ 其他___
```

### 用戶畫像（D）
```
您的職業？教學年資？團隊規模？
```

## 問卷設計的三個地雷

1. **避免「假設性題目」**
   - ❌ 「如果我們有這個功能，您會付費嗎？」
   - ✅ 「過去一年，您曾為類似工具付費過嗎？金額？」

2. **避免「滿意度量表」當主軸**
   - ❌ 「您對現況滿意度是 1-5 分中的幾分？」
   - ✅ 改成挖代價、挖頻率、挖具體解決方案

3. **避免引導性題目**
   - ❌ 「您是否同意 OO 是一個重要的問題？」
   - ✅ 「請描述上次遇到 OO 的情境與處理方式」

---

## VJPD 自檢清單

寫完訪談大綱 / 問卷後，用這份清單檢查：

- [ ] V 的題目至少 2 題，且都鎖定具體事件
- [ ] J 的題目至少 2 題，且能量化（時間 / 金錢 / 後果）
- [ ] P 的題目至少 2 題，且追問了「為什麼還在用現有方案」
- [ ] D 的題目放在後段，不會搶開頭注意力
- [ ] 沒有任何題目用「會不會」「願不願意」「覺得」開頭
- [ ] 沒有任何題目要求對方「想像」未來情境
