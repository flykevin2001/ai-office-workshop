# 看板管理

## 核心概念

看板就是一份 Obsidian 的 .md 檔案，搭配 Kanban 外掛使用。
沒裝外掛也能用，只是顯示為普通的 Markdown 勾選清單，功能不受影響。

Kanban 外掛來源：https://github.com/mgmeyers/obsidian-kanban

### 看板的本質

看板是你的「工作速查表」。AI 一打開就知道你現在在忙什麼。
打勾的就是做完的，沒打勾就是還沒做的。就這麼簡單。

### 看板的位置

預設放在 `00 工作台/🎯 工作看板.md`。
這是整個 AI 辦公室最重要的檔案之一——AI 每天第一個讀的就是它。

---

## 看板檔案格式

檔案開頭需要這段（讓 Kanban 外掛認得）：

```yaml
---
kanban-plugin: board
---
```

每一列代表一天，用二級標題：

```markdown
## 04-16 三
```

列底下放卡片，用勾選格式：

```markdown
## 04-16 三

- [ ] 09:00 開會
- [ ] 整理筆記
- [x] 回覆信件
```

檔案結尾可能有 Kanban 外掛的設定區塊（`%% kanban:settings ... %%`），操作時保留不動。

---

## 卡片格式

四種寫法，依情境選用：

| 情境 | 格式 |
|------|------|
| 有時間 | `- [ ] HH:MM 事項名稱` |
| 沒時間 | `- [ ] 事項名稱` |
| 有對應筆記 | `- [ ] [[筆記連結\|HH:MM 事項名稱]]` |
| 已完成 | `- [x] 事項名稱` |

原則：
- 卡片只寫「時間 + 事項名稱」，保持精簡
- 詳細資訊寫在對應的筆記檔裡，卡片只放連結
- 不確定的事也要先開卡，標上 `#尚未確認`

## 三種操作

### 開卡

觸發詞：「開卡」「加到看板」「記一下」「幫我排進去」

流程：
1. 讀取工作看板
2. 判斷放哪一天（使用者有說日期 → 放對應日期；沒說 → 問一句）
3. 該日期的列不存在 → 新建一個，按日期順序插入
4. 先列出即將寫入的內容，等使用者確認
5. 確認後寫入

使用者一次講多件事 → 每件事各開一張卡。

**快速開卡**：使用者說「快速開卡：XXX」→ 直接寫入，不追問，預設今天。這是唯一不需要確認的例外。

### 查看板

觸發詞：「今天做什麼」「看板狀態」「最近有什麼」

流程：
1. 讀取看板
2. 找到今天（或指定日期）的列
3. 分兩區回報：還沒做的、已完成的
4. 有過期未完成的卡片 → 主動提醒
5. 用自然語言回報，不貼 Markdown 原始碼

### 整理看板

觸發詞：「整理看板」「看板太長了」

流程：
1. 找出超過 30 天且全部完成的列 → 建議備存
2. 有未完成卡片的舊列 → 問使用者怎麼處理
3. 列出整理計畫，確認後執行
4. 備存到 `00 工作台/看板備存/工作看板 YYYY-MM.md`

核心：**不刪除，全部備存。未完成的一定要問。**

---

## 操作原則（所有操作都適用）

- 所有動作先建議、等確認、再執行
- 不確定的事也開卡（先記下來比忘掉好）
- 卡片保持精簡，詳細內容放筆記
- 不刪除卡片，舊的備存起來
- 寫入時保留 Kanban 外掛的設定區塊
- 不動使用者的其他檔案，只處理看板相關的

## 四種看板應用範例

上面的「工作看板」是核心概念。同樣的機制，可以變化成不同的應用。
以下是四種常見的看板用法，你可以根據自己的需求選擇性建立。

### 應用一：工作看板（預設）

追蹤日常任務和專案進度。一列 = 一天，每天 5-10 張卡片。

```markdown
## 04-16 三

- [ ] 09:00 團隊週會
- [ ] 整理上週報告
- [ ] 回覆合作邀約
- [x] 寄出發票
```

適合：所有人。這是最基礎的用法，初始化時就會建好。

### 應用二：發文看板

追蹤社群貼文的生命週期。一列 = 一天或一週，卡片是每篇貼文。

```markdown
## 04-16 三

- [ ] 14:00 ｜ [[AI 辦公室心得|AI 辦公室三個月心得]]
- [x] 10:00 ｜ [[工具推薦|推薦三個 Obsidian 外掛]]
```

適合：有在經營社群的人。把「寫文 → 排程 → 發布 → 歸檔」的流程視覺化。

### 應用三：邀約看板

追蹤跟人有關的約定。一列 = 狀態（發想中 / 已邀約 / 已確認 / 已完成）。

```markdown
## 已確認

- [ ] 04-20 ｜ 跟 Andy 聊合作方向
- [ ] 04-25 ｜ 客戶 Demo

## 已邀約

- [ ] 等回覆 ｜ 邀約王先生下週線上聊
```

適合：需要管理多個合作關係或會面行程的人。

### 應用四：課程/專案看板

追蹤較大的時間跨度。一列 = 一個月或一個階段。

```markdown
## 四月

- [ ] 04-12 ｜ AI 辦公室基礎班 第一梯
- [ ] 04-26 ｜ AI 辦公室基礎班 第二梯

## 五月

- [ ] 05-10 ｜ 進階班 第一梯
```

適合：有在開課、辦活動、管長期專案的人。

---

## 如果你想用多個看板

1. 每個看板是一份獨立的 .md 檔
2. 都放在 `00 工作台/` 裡
3. 命名建議：`🎯 工作看板.md`、`📝 發文看板.md`、`🤝 邀約看板.md`、`📅 課程看板.md`
4. 跟 AI 說「查工作看板」「查發文看板」，AI 就知道要看哪一份

江江教練自己同時用四個看板，但建議新手先從一個「工作看板」開始。等你覺得一個看板不夠分類了，再拆就好。
