# 標籤管理技能包 1.0

幫使用者從零建立、維護自己的標籤系統。核心精神：所有動作都先建議、等確認、再執行。

## 字典檔位置

- 預設路徑：與此技能包同資料夾下的 `tag-dictionary.md`
  - 實際路徑：`_agent/skills/tag-manager v1.0/tag-dictionary.md`
- 找不到時，先問使用者是否已有字典檔、放在哪裡
- 第一次使用且確認沒有字典檔時，用同資料夾的 `tag-dictionary-template.md` 建立一份

## 標籤格式

```
#標籤名稱
```

放置位置：文件主標題後面獨立一行，標籤行之後空一行再接內容。

範例：
```
# 文件標題

#Python學習 #提示詞工程 #RAG

正文內容從這裡開始……
```

## 功能一：自動建議標籤

觸發語：「幫我標標籤」「這篇該用什麼標籤」「幫我想標籤」

### 流程

1. 檢查是否有 `tag-dictionary.md`
   - 有 → 讀取，後續建議優先從字典裡選
   - 沒有 → 直接根據文件內容建議（同時提醒使用者可以建字典檔，問要不要現在建）

2. 讀取使用者提供的文件內容，理解主題

3. 建議 3-5 個標籤，分兩類呈現：

   **字典檔已有的詞**（直接建議）：
   - `#Python學習`
   - `#提示詞工程`

   **字典檔沒有的詞**（標記為新詞）：
   - `#RAG` ← 新詞，要收錄嗎？

4. 等使用者確認或調整後，才把標籤寫入文件

### 建議標籤的原則

- 要具體，不要太抽象（「學習」太泛，「Python學習」比較有用）
- 同一件事只用一個詞，發現同義詞要提醒統一
- 想一下「未來的自己會用什麼詞搜尋這份文件」——標籤是為了找回來用的
- 新詞累積超過 10 個時，主動提醒使用者啟動功能二來整理

## 功能二：整理受控詞彙

「受控詞彙」的意思：同一個概念，只用同一個詞。不讓「AI」「人工智慧」「Artificial Intelligence」三個詞同時存在於知識庫。

觸發語：「整理標籤」「整理受控詞彙」「標籤太亂了」
自動觸發：功能一累積超過 10 個新詞時主動提醒

### 流程

1. 請使用者提供近期標註過的檔案清單（或掃描知識庫資料夾）

2. 列出所有出現過的標籤與使用次數，例如：
   ```
   #AI (12次) #人工智慧 (3次) #Artificial-Intelligence (1次)
   #提示詞 (8次) #Prompt (5次) #提示詞工程 (2次)
   #RAG (1次)
   ```

3. 找出三種問題並分類：
   - **同義詞**：同一件事用了不同的詞
   - **低頻詞**：只出現過一次的標籤（可能太細，或是一時的用詞）
   - **未收錄詞**：有在用、但不在字典檔裡的標籤

4. 一次討論一組，從影響最大的同義詞開始：
   - 「`#AI` 和 `#人工智慧` 是同一件事嗎？要統一用哪一個？」
   - 「`#RAG` 只出現一次，要保留還是刪除？」
   - 「`#提示詞` 這個新詞要正式收錄嗎？」

5. 使用者確認後，自動接續功能三寫入字典檔

### 討論原則

- 一次只討論一組，不要一口氣丟太多選擇給使用者
- 先處理同義詞（影響最大），再處理低頻詞
- AI 給建議，使用者做決定

## 功能三：寫入字典檔

觸發語：「加到字典」「新增標籤」
自動接續：功能二確認後

### 流程

1. 讀取現有 `tag-dictionary.md`
   - 沒有字典檔 → 用 `tag-dictionary-template.md` 建一份，建完再繼續

2. 把新詞寫入對應分類下
   - 使用者還沒建分類 → 先問要放哪一類，或建議新分類，等確認後再寫

3. 如果是合併同義詞：
   - 字典檔裡把被合併的詞移到「已停用詞彙」區，並註明改用哪個詞
   - 列出哪些舊檔案需要手動更新標籤（只列清單，不自動修改文件）

4. 寫入完成後回報：
   - 新增了什麼
   - 放在哪一個分類下
   - 有沒有需要手動更新的舊檔案

---

## 不做的事

- 不改檔名
- 不整理文件內容，只處理標籤
- 不在使用者沒確認的情況下寫入任何標籤或修改字典檔
- 不自動批次修改舊文件的標籤（只列清單，讓使用者自己決定）

---

## 字典檔格式說明

字典檔分三個區塊：

1. **受控詞彙表** — 正式收錄的標籤，依分類整理
2. **已停用詞彙** — 合併同義詞後被取代的舊詞，保留紀錄備查
3. **待確認新詞** — 功能一發現但使用者還沒決定要不要收錄的詞

詳細格式見同資料夾的 `tag-dictionary-template.md`。
