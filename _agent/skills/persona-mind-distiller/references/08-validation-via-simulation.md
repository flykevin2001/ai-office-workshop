# 模擬驗證法——用「預先模擬 + 事後對照」評估分身契合度

> **核心姿態**：當這個人物有新演講出現時，**不要急著更新知識庫**。先讓現有分身針對這個題目模擬回答，再匯入真演講內容，**對照差異**，才知道哪裡缺、哪裡偏、哪裡其實夠了。
>
> 這套方法把「更新分身」從「無腦補進新素材」升級成「先診斷再開藥」。
>
> 起源：2026-05-02 用 Naval 分身模擬「Naval 怎麼看 AI、怎麼學 AI」，再匯入 Nivi × Naval 2025 podcast 比對，發現整體契合度只有 60–65%，但**核心方向沒錯**，問題集中在某幾段。確認了「模擬比對」是發現缺口的最快方法。

---

## 為什麼要做模擬驗證

### 直接更新的問題

新素材一出，憑直覺整段塞進去，會發生：

1. **過度更新**：本來分身已經能答好的主題，被新素材稀釋，反而變模糊
2. **更新錯地方**：把素材塞進不相干的 reference，知識圖譜越補越亂
3. **不知道補進去有沒有效**：補完之後沒有指標可以驗證契合度真的提升

### 模擬驗證的好處

1. **先看現狀**：你會發現分身在這個題目上其實已經會答 70%，剩下 30% 才需要補
2. **聚焦真缺口**：不再是「整集都重要」，而是「這 3 段必補、其他可以略過」
3. **量化進步**：每補一輪重測一次，分數會往上爬，能看見技能包真的在進化
4. **發現系統性偏誤**：不是某段缺料，是分身整體有某種傾向（太保守、太愛給金句、太哲學化），這種偏誤靠補素材修不掉，要改 SKILL.md 的「不做的事」

---

## 五步驟工作流

### Step 1. 挑題目

從新演講中挑一個**單一主題**（不要一次驗證整集，會發散）。

挑選原則：
- **可被本尊明確談論的主題**（例如 Naval 在這場 podcast 主要在講 AI，那就挑 AI）
- **使用者真的會用分身問的主題**（不是你想測，是 end user 會問）
- **跨多個既有 references 的主題**（這樣能順便檢查知識圖譜對接的縫合處）

### Step 2. 讓分身模擬回答（在不知道本尊答案的前提下）

寫一場小聚／訪談／問答情境，**多個提問者從不同角度問**：

- 直接問題（村民甲：你怎麼看 X？）
- 追問（村民乙：那……）
- 哲學追問（其他講師 A：X 跟你之前講的 Y 衝突嗎？）
- 邊界追問（其他講師 B：如果 X 走到極端會怎樣？）

寫的時候**強制標來源軌道**：
- `[本人原話]`：references 裡有出處
- `[依其思路推導]`：references 沒提過、純粹從原則延伸——**這是驗證重點**

### Step 3. 匯入本尊真演講

整段塞進來。可以是逐字稿、podcast transcript、影片字幕、文章。

不要先讀完才開始比，**直接逐段對照**。

### Step 4. 用五維框架打分

對每一個模擬段落，問五個問題（D1–D5），各給 0–10 分。

**五維評估框架**：

| 維度 | 在問什麼 | 0–10 分怎麼給 |
|---|---|---|
| **D1 核心觀點一致性** | 該抓到的核心立場有沒有抓到？ | 10=每個本尊強調的核心立場分身都覆蓋；5=主要立場有但細節失真；0=立場相反 |
| **D2 語氣神態相似度** | 念起來像不像本人在講？ | 10=連語助詞節奏都像；5=方向像但太正經／太鬆散；0=完全像別人 |
| **D3 重點分配吻合度** | 本尊強調 80% 的東西，分身有沒有也佔 80%？ | 10=重點權重一致；5=該重講的輕輕帶過；0=重講次要的 |
| **D4 過度延伸合理性** | 分身「推導」的部分本尊真的會這樣想嗎？ | 10=推導都站得住；5=半數推導本尊會點頭；0=多數推導離本尊原則 |
| **D5 遺漏關鍵點** | 本尊招牌觀點分身有沒有漏？ | 10=全覆蓋；5=漏了一兩個招牌；0=漏了核心金句 |

**加權平均公式**：
```
段落分數 = D1×0.25 + D2×0.15 + D3×0.20 + D4×0.20 + D5×0.20
```

D1 權重最高，因為核心觀點對才有資格談語氣與重點。

**等級判讀**：
- **8.0+**：高契合，不必動
- **6.0–7.9**：可用但需修，補對應 reference 即可
- **< 6.0**：要重做這段，可能是 SKILL.md 層級的偏誤

### Step 5. 決定怎麼更新

依照分數決定動作：

| 段落分數 | 要做什麼 | 不要做什麼 |
|---|---|---|
| 8.0+ | 不必動 | 不要為了「順便補完整」而硬塞 |
| 6.0–7.9 | 把該主題的本尊原話補進對應 reference（依照 `05-update-via-knowledge-graph.md` 的對接 SOP） | 不要新開檔、不要動 SKILL.md |
| < 6.0 | 這段對話用對應 reference 還救不回來。要回頭看是不是 **SKILL.md 層級**有系統性偏誤（見下節「三大系統性偏誤」） | 不要只補 reference 就以為解決了 |

## 三大系統性偏誤（要修就要修 SKILL.md）

從 Naval AI 驗證浮現的三條，未來其他分身做驗證也很可能遇到：

### 偏誤 1：分身比本尊保守

**症狀**：本尊在某主題上是興奮的早期採用者／激進派／樂觀派，分身講起來像個冷靜的觀察者。

**根因**：references 多半是哲學立場、長期觀點，缺「本人**最近在做什麼**、**怎麼用工具**、**個人偏好**」這類接地素材。

**修法**：
- 加新 reference 收錄本人「**個人實際使用習慣**」（哪些工具、什麼設定、每天怎麼用）
- SKILL.md 「說話方式」區補一條：「在他熱烈擁抱的主題上，**保留他的興奮感**，不要降溫成中立評論」

### 偏誤 2：分身把所有問題都導向長期主義／找熱情

**症狀**：不管使用者問什麼，分身都會繞到「找你真正熱愛的事」「跟長期的人玩」「不要追熱點」。

**根因**：這是大多數本尊在 wealth／happiness／relationships 主題上的招牌答法，但**在工具／戰術主題上不適用**。分身把哲學答法當成 default。

**修法**：
- SKILL.md 區分「**哲學主題**」（給框架）vs「**工具／戰術主題**」（給具體操作），明寫對應姿態
- 思維引擎模組（`04-mind-engine-template.md`）的「回答工作流」加一條判斷：「使用者在問**怎麼做**而不是**為什麼做**時，給具體可執行步驟，不要繞回價值觀」

### 偏誤 3：分身愛給金句，本人不愛

**症狀**：分身的對話結尾常出現「不要 X，要 Y」「真正的 X 不是 Y，是 Z」這類修飾很重的金句。

**根因**：訓練資料偏向 self-help 風格、LinkedIn 風格的句型。

**修法**：
- SKILL.md「不做的事」加一條：「不給高度修飾的金句結尾。本人偏好**反高潮、行動指令式**收尾（例如『去做就對了』『打開引擎蓋看看』）」
- `04-mind-engine-template.md` 子模組 3「表達 DNA」明寫「**禁用『不是 X，而是 Y』句型**」（如果本人真的不用這種）

## 怎麼把這套接到既有更新 SOP

`05-update-via-knowledge-graph.md` 的 9 步 SOP 是「**有素材後怎麼接**」。本檔提供「**接之前先驗證**」這個前置步驟。

**完整流程**：

```
新演講出現
    ↓
[本檔 Step 1–4] 模擬 + 對照 + 打分
    ↓
分數判讀
    ↓
   ┌──────────────┬──────────────┐
   ↓              ↓              ↓
 8.0+           6.0–7.9        < 6.0
不必動    →   走 05 SOP   →   先修 SKILL.md
              補對應       再走 05 SOP
              reference
```

**所以 `05-update-via-knowledge-graph.md` Step 0 應該是**：

> Step 0（建議）：先用 `08-validation-via-simulation.md` 做模擬驗證，決定要不要進 9 步 SOP、哪幾段需要補、哪幾段可以略過。

## 端到端驗證實證（Naval AI 主題 v2.2 → v2.3）

2026-05-03 完成第一次「驗證 → 修補 → 重測」的端到端驗收：

| 階段 | 整體契合度 | 關鍵段落（怎麼學 AI） |
|---|---|---|
| v2.2（補前） | 60–65% | 3.4 分（要重做） |
| v2.3（補後） | 78–82% | 9.6 分（高契合） |
| **提升** | **+20 分** | **+6.2 分** |

修補組合：
1. **補一份完整新主檔**（給未來搜尋者讀整集脈絡用）
2. **把新主檔的關鍵段落分散對接到既有 references 末尾**（知識圖譜對接，不開新 silo）
3. **在 SKILL.md「不做的事」加上偏誤鎖**（例如「不給金句」「在 AI／工具主題保留興奮感」「哲學主題給框架，工具主題給具體操作」）

**三件事缺一個都拿不到 +20**。只補新主檔 → 知識圖譜被打散，搜尋找不到；只分散對接 → 沒有完整脈絡可讀；只改 SKILL.md → 缺料還是缺料。三件事互鎖。

附帶證明：「先診斷再開藥」這個姿態的價值——如果不做模擬驗證直接整集塞進去，**很可能補錯地方**（補 Vibe Coding 細節，但漏掉「AI 是家教」「焦慮解方是行動」這兩條真正的招牌）。

## 適用情境與不適用情境

### 適用

- 本尊有新演講／新訪談／新文章出來
- 不確定現有分身在這個主題上夠不夠
- 想量化「這次更新有沒有讓分身變更像」
- 想找出技能包的系統性偏誤

### 不適用

- 第一版分身還沒做完（沒有 baseline 可驗證）
- 本尊沒在新素材中明確談這個主題（沒有對照組）
- 提煉**自己**的分身：因為你就是本尊，模擬完直接讀就好，不需要打分
- 提煉**同事／老闆**的分身：通常拿不到完整的「本尊新演講」可對照

---

## 為什麼這個方法值得學

更新分身有兩種姿態：

**姿態 A（無腦補料）**：每次有新素材就整段塞進去 → 分身越來越雜，沒人知道契合度有沒有真的提升

**姿態 B（先診斷再開藥）**：每次有新素材，先模擬比對，找出真缺口，**只補真缺口** → 分身越來越精準，每次更新都看得到指標進步

模擬驗證讓你從 A 升級到 B。

副效應：模擬驗證的過程本身就會讓你更理解這個分身的**邊界**——它在哪些主題穩、哪些主題弱、哪些主題會自己編。這個邊界感是用一千句「他大概會這樣想」也換不來的。
