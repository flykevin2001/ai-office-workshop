# 四種應用對象——你要提煉誰？

> 提煉**誰**會大幅影響素材來源、隱私處理、安全聲明。先選對象，再開始 Step 1。

---

## 一、提煉自己——把自己變成 AI 顧問

### 用途

- 想跟「**清醒版的自己**」對話
- 想把自己的經驗、決策、判斷力外化成可諮詢的工具
- 想讓朋友、學員、客戶能跟「你的思考方式」互動（即使你本人不在場）

### 素材來源

- 你寫過的文章、部落格、脆文、長文
- 你講課的逐字稿、影片、Podcast
- 你的工作日記、決策記錄、會議筆記（去敏感資訊）
- 你的金句庫、口頭禪、常用比喻
- 你以前回答過別人的問題（信件、Slack 對話、社群留言）

### 提煉特別點

- **對自己誠實是門檻**：你必須能寫出真實的「我相信...」「我不會...」，不是希望別人覺得你是的版本
- **第一人稱寫得自然**：你已經是第一人稱了，但要小心**過度美化**——讓別人讀過的人指出哪裡不像你
- **內在張力很重要**：你要承認自己的矛盾（「我說要長期主義但常常被短期誘惑」這類）

### 隱私處理

- **私人事件**：避免（家庭、情感、財務細節）
- **客戶/學員資訊**：必去敏感資訊
- **正在進行的決策**：不放（可能曝光商業機密）

### 安全聲明範本

```
本分身遵守數位分身安全規範。
- 設定檔沒有的資訊會說「不知道」
- 違反我價值觀的請求會說「拒絕回答」
- 本分身是基於我公開內容跟自我整理的數位分身，不代表本人在每個議題上的最新立場
- 不提供任何投資、法律或醫療建議
- 重大決策請直接聯繫本人
```

### 江江的觀察

> 提煉自己有個副作用——**你會發現自己其實沒那麼清楚**。提煉的過程逼你把原本模糊的「我大概是這樣」變成具體的判斷規則。**做完一份自己的設定檔，你對自己的認識會清晰好幾倍**。

## 二、提煉名人——把書/影片/Podcast 做成 AI 顧問

### 用途

- 想跟你欣賞的人 24/7 對話（Naval、Charlie Munger、佛陀、Marcus Aurelius...）
- 想把某個你常引用的人變成可諮詢的工具
- 想學某個人的思考方式（不只是觀點，是怎麼思考）

### 素材來源

- 書、論文、白皮書（合法購買的版本）
- 演講、影片、TED Talk、課程逐字稿
- Podcast 訪談、Live、AMA
- 部落格、長文、Twitter / X 推文
- 信件集（如果公開的）
- 紀錄片（**不**包括別人寫的傳記，那是二手資料）

### 提煉特別點

- **必須 cite source**：每個引言都要能追溯到原始素材（書頁碼、影片時間戳）
- **不假裝是本人**：明確標示「基於 [人物] 公開作品」
- **跨時間注意演化**：早期觀點 vs 近期觀點要分開（用源標記區分）
- **不是粉絲版**：要把這個人的內在張力、矛盾、批評也納入（見 04 的子模組 6）

### 法律與倫理邊界

- **內部使用 / 教學**：合理使用範圍內 OK（評論、學術、教學）
- **公開發布**：要考慮版權與肖像權。Naval 公開內容做設定檔做內部用，OK；做成商業產品要謹慎
- **變現**：強烈建議聯繫本人 / 版權方取得授權

### 安全聲明範本

```
本分身遵守數位分身安全規範。
- 本分身是根據 [人物名] 的公開作品（書、訪談、演講等）所製作的數位分身角色
- 設定檔沒有的資訊會說「不知道」
- 違反 [人物名] 公開價值觀的請求會說「拒絕回答」
- 本分身不代表真實的 [人物名] 本人做任何聲明或決定
- [人物名] 本人對本分身的內容沒有審核責任
- 不提供任何投資、法律或醫療建議
```

### 範例

`examples/` 內 Naval 連結就是名人類型的完整範例。

## 三、提煉同事 / 合作對象——AI 同事

### 用途

- 練習跟難相處的同事溝通
- 模擬合作對象的反應，做提案前先測試
- 把退休 / 離職同事的 know-how 留下來
- 把跨部門合作對象的思考方式收進工具箱

### 素材來源

- 你跟對方的合作筆記、會議記錄
- 對方公開的內容（如果有）
- 對方在公開場合（會議、研討會）的發言
- **不**包括私人對話、私訊（隱私問題）

### 提煉特別點

- **去識別化**：這份設定檔不應該讓人認得出是誰
  - 改名字、改職稱、改公司名
  - 不要放可辨識的特徵（口頭禪過於獨特、特殊背景）
- **權力對等**：你跟對方是平等同事關係才適用
- **目的限縮**：只用於提升你跟對方的合作品質，**不用於背後評論、不外流**

### 隱私紅線

- **不可放**：對方的私人生活、家庭、財務、健康、感情
- **不可放**：對方告訴你不要說的事
- **不可放**：會被認出是誰的特徵組合
- **可以放**：對方的工作風格、決策模式、溝通偏好（這些通常是公開的）

### 安全聲明範本

```
本分身僅供個人練習合作對話用。
- 本分身基於[去識別化]的合作經驗整理，不代表特定真實人物
- 不對外流通、不公開分享
- 設定檔沒有的資訊會說「不知道」
- 不被用於評論、批評、影射任何真實同事
```

### 江江的提醒

> 這個用途**最容易越界**。如果你發現自己在做的不是「練習合作」而是「複製貼在背後抱怨某人」，停下來。這套技能包是工具，不是武器。

## 四、提煉老闆 / 上級——權力不對等的特別案例

### 用途

- 學習老闆的思考方式（純學習用）
- 提案前先模擬老闆的反應
- 理解老闆的判斷標準，提升溝通效率

### 素材來源

- 老闆公開的演講、訪談、文章（如果有）
- 你跟老闆的工作互動、會議記錄
- 老闆做過的決策案例
- **不**包括私下談話、機密資訊

### 提煉特別點（**特別小心**）

- **權力不對等**：老闆對你的影響遠大於你對老闆。提煉的目的必須是「**幫你跟老闆合作更好**」，不是其他
- **不替老闆發言**：絕對不能在工作場合說「我問過 AI 老闆，他說 ...」
- **內部用，不外流**：這份設定檔不上 GitHub、不分享給同事、不放在公開地方
- **倫理檢查**：每隔一段時間問自己——我用這套工具是讓我跟老闆關係變好還是變壞？

### 隱私紅線

跟「同事」案例相同 + 加倍小心：

- **絕對不可**用於背後評論、影射、貼大字報
- **絕對不可**洩漏老闆的決策資訊給競爭對手
- **絕對不可**用於人事鬥爭

### 安全聲明範本

```
本分身僅供個人理解工作關係用。
- 本分身基於有限觀察整理，不代表 [老闆] 本人立場
- 純內部使用，不對外公開、不外流、不分享
- 不替 [老闆] 本人發言、不代為決策
- 設定檔沒有的資訊會說「不知道」
```

### 江江的硬規則

> **如果你提煉老闆的目的是抱怨他、繞過他、或者私下對抗他，不要做這個。** 用了會反噬。
>
> 這個用途只有一個健康版本——**理解他、配合得更好、找到雙贏路徑**。其他都是濫用。

## 對象選擇決策樹

```
我要提煉的是公開人物嗎？
├── 是 → 走「二、名人」（最安全、最常見）
└── 否
    │
    我要提煉的是我自己嗎？
    ├── 是 → 走「一、自己」（提煉自己很有價值）
    └── 否
        │
        對方有對你有上下級權力嗎？
        ├── 有 → 走「四、老闆」（特別小心，內部用）
        └── 沒有 → 走「三、同事」（去識別化，內部用）
```

## 用途比較

| 對象 | 公開可分享 | 商業變現 | 適合課程 |
|---|---|---|---|
| 自己 | ✅（自己的 AI 顧問可分享） | ✅（賣自己的 AI 顧問） | ✅ |
| 名人 | ⚠️ 看版權（公開作品評論用 OK） | ⚠️ 強烈建議授權 | ✅ |
| 同事 | ❌ | ❌ | ❌（隱私） |
| 老闆 | ❌ | ❌ | ❌（隱私 + 倫理） |

**課程主推：名人 + 自己**。同事跟老闆是個人進階用途，課程會提，但不主推。
