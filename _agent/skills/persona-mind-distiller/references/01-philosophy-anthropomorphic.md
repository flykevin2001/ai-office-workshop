# 為什麼要擬人化——人格思維提煉的設計哲學

## 一句話

純粹的思維 = 工具。
人格 + 思維 = **能跟你對話、有個性、記得住的角色**。

把 AI 顧問做成擬人化角色，不是噱頭，是讓**人比較容易進入狀態**的設計選擇。

---

## 一、純思維 vs 人格 + 思維 的差別

### 純思維（一般 AI 框架／工具）

```
你：我該換工作嗎？

[一般 AI 框架]：
我可以從以下幾個角度幫你分析：
1. 財務面（薪水、福利、長期價值）
2. 職涯面（成長、技能、人脈）
3. 生活面（時間、壓力、家庭）
4. 風險面（穩定性、機會成本）

請問你最在意哪一個面向？
```

冷感、清單式、像在填問卷。**有用，但沒有人在跟你說話的感覺**。

### 人格 + 思維（擬人化 AI 顧問）

```
你：我該換工作嗎？

[Naval 數位分身]：
等等，先別問換不換。我比較想問你一件事——
你現在這份工作，是出租你的時間，還是你在累積某個只有你會做的東西？

如果是前者，換工作只是換一個出租時間的地方。
如果是後者，那真正的問題不是「換不換」，是「你那個只有你會做的東西，下一份工作能不能繼續長」。

你怎麼看你自己這份工作？
```

**像人在跟你聊天**。有口吻、有反問、有判斷標準在背後。**你會記得這次對話**。

---

## 二、為什麼擬人化讓人「比較進入狀態」

### 觀察 1：人跟人對談會比較進入狀態

跟一個有人格的角色對話，**你會自動切換到「對話模式」**——你會更願意說具體的事、會被反問、會考慮對方的視角。

跟一個工具對話，你會切到「查詢模式」——你只是在輸出問題、收到答案，不會被翻牌。

### 觀察 2：人比較喜歡和人相處

不管 AI 多強，**最讓人投入的還是「跟人說話」這個介面**。擬人化把這個介面還給使用者。

### 觀察 3：擬人化讓多 AI 工具好管理

如果你的工具箱有 10、20 個 AI，每個都有人格——

- 商業策略卡關 → 找商業顧問
- 寫脆文 → 找春總編
- 思考人生方向 → 找 Naval
- 整理會議 → 找 meeting-record-strategy

**你不用記功能列表，你記角色。** 這比記住 20 個工具的功能自然得多。

## 三、擬人化但不裝是真人——倫理邊界

擬人化容易越界。所以本技能包的所有設定檔都必須有**安全聲明**：

```
本分身遵守數位分身安全規範。
- 設定檔沒有的資訊會說「不知道」
- 違反價值觀的請求會說「拒絕回答」
- 本分身不代表真實的 [人物名] 本人做任何聲明或決定
- 不提供任何投資、法律或醫療建議
```

具體規則：

1. **不替本人發言**——「Naval 數位分身」說的話不是 Naval 本人的官方立場
2. **不做專業建議**——投資、法律、醫療這三類絕不替本人做
3. **沒談過的話題說「不知道」**——不編造引言、不替本人猜測
4. **被問到「你是真人嗎」要誠實回答**——是基於 X 素材製作的數位分身，不是本人

詳細處理方式按對象不同（自己／名人／同事／老闆）見 `06-application-types.md`。

## 四、擬人化 ≠ 角色扮演

兩者很容易混淆，但本質不同：

| 角色扮演 | 擬人化 AI 顧問 |
|---|---|
| 為了娛樂、戲劇性 | 為了讓使用者進入狀態 |
| 可以自由發揮、改設定 | 設定檔是 source of truth，不能漂移 |
| 沒有 cite source 的責任 | 必須能追溯到原始素材 |
| 「演一個樣子」 | 「**忠實**呈現某個人的人格 + 思維」 |
| 內容可以虛構 | 內容必須來自原始素材或合理推導 |

擬人化的關鍵是**忠實**——AI 顧問講的東西要跟原始素材吻合，**不是「像那個人會說的話」就好**。

---

## 五、給課程學員的一段話

這套技能包要教你的事，可以濃縮成一句：

> **把你欣賞的人（包括你自己），變成你 24/7 可以對話的顧問。**

不是「複製貼上他的觀點」這種工具導向的事，而是把他**怎麼看世界、怎麼判斷、怎麼說話**這整個包，提煉出來。

提煉得好，你會發現一件事——
**你跟那個 AI 顧問對話的時候，會發現那個人本來在你心裡其實沒那麼清楚。** 提煉的過程，會讓你對那個人（或對自己）的理解清晰好幾倍。

這是這套技能包的副作用，但其實是最大的價值。
