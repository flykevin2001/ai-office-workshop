# 知識圖譜對接更新 SOP——新訪談集／新觀點怎麼接進去

> 你做完 v1.0 之後，那個人會繼續產出新內容（出新書、新訪談、新觀點）。
>
> **錯誤做法**：每次都建一個獨立檔案，最後變成一堆孤立 silo。
>
> **正確做法**：按主題拆散到既有 references。這樣讀者搜「槓桿」這個主題會自動拿到 v1.0 + v2.0 + v3.0 的累積觀點。
>
> 這條 SOP 從 Naval 技能包跨集對接的實戰浮現出來。9 步走完。

---

## 兩種更新姿態

### 孤立附件（錯）

```
references/
├── 01-leverage.md
├── 02-thinking.md
├── 03-naval-2025-interview.md   ← 新訪談集獨立成檔
├── 04-naval-2026-interview.md   ← 又一集獨立成檔
└── ...
```

問題：讀者搜「槓桿」時不會看到 03、04 裡的最新觀點，因為它們是孤立 silo。

### 知識圖譜對接（對）

```
references/
├── 01-leverage.md
│   ├── 原本的書本內容
│   ├── ## 2025 訪談延伸：純軟體不可投資
│   └── ## 2026 訪談延伸：個人 App Store
├── 02-thinking.md
│   ├── 原本的書本內容
│   └── ## 2025 訪談延伸：AI 取悅你
└── 01g-vibe-coding-2026訪談.md  ← 整集主檔（給想看完整脈絡的人）
```

每個訪談集**按主題拆散**到既有檔案末尾，加「## YYYY 訪談延伸」段落。同時保留一份整集主檔給想看脈絡的人。

## 9 步 SOP

### Step 0（建議）：先做模擬驗證再決定要不要進 SOP

新素材一出，**不要直接動手補**。先用 `08-validation-via-simulation.md` 的方法做一輪模擬比對：

1. 從新素材挑一個主題
2. 讓現有分身針對這題模擬回答（不看新素材）
3. 匯入新素材逐段比對
4. 五維打分（D1–D5）

**判讀**：
- 整體 8.0+ → 不必進這套 SOP，分身已經夠
- 6.0–7.9 → 進 9 步 SOP，但只補低分段落對應的 reference
- < 6.0 → 先回頭看是不是 SKILL.md 層級有系統性偏誤（保守、哲學化、愛給金句），修完 SKILL.md 再進 SOP

這個前置步驟讓更新從「無腦補料」升級為「先診斷再開藥」。

---

### Step 1. 抓日期、定來源標記

新訪談集進來，第一件事：

- **確切日期**（YouTube：用 `curl + grep '"uploadDate"'` 抓）
- **定來源標記**：
  - 第一集：`[書]` 或 `[2025訪談]`
  - 第二集：`[2025訪談II]` 或 `[2026訪談]`
  - 編號規則：年份 + 序號（同年第二集就 II、III、IV）

### Step 2. 盤點主題 → 決定對接到哪幾個既有 references

讀完整集，列一張**對接表**：

| 章節主題 | 對接到既有檔案 | 橋接邏輯（這集怎麼跟既有原則接上）|
|---|---|---|
| 純軟體不可投資 | `01c-leverage` | 「程式碼槓桿」的最新極致 |
| AI 太想取悅你 | `02a-clear-thinking` | 「自我看不見現實」的 AI 時代驗證 |
| ... | ... | ... |

通常 5-8 個對接點。少於 3 個表示這集太薄；多於 10 個表示你還沒分清主題。

### Step 3. 建主檔（含對接表）

新檔案命名格式：`01X-集名-YYYY訪談N.md`（例 `01g-Vibe-Coding-2026訪談.md`、`01h-In-the-Arena-2025訪談II.md`）。

主檔結構：

```markdown
# 集名 — YYYY 年訪談 N

> 話題涵蓋：[簡述]
>
> **這集是 [人物] 對 [核心既有原則] 的最新延伸。** 各章節對接到既有檔案：
>
> | 既有檔案 | 對接章節 |
> |---|---|
> | `01c-...` | 第 X 章（XXX）|
> ...

---

## 來源
[YYYY訪談N] = ...

完整章節時間軸：...

---

## 一、[章節 1]
[2025訪談II]
...
```

主檔是「整集精華 + 章節對接表」，給想看脈絡的人。

### Step 4. 既有檔末尾加延伸段

對應到的每個既有檔案末尾加：

```markdown
---

## YYYY 訪談 N 延伸：[標題] `[YYYY訪談N]`

> 詳見 `01X-集名-YYYY訪談N.md` 第 X 章。[1 句橋接邏輯——這集怎麼跟原本原則接上]

### [小段標題 1]
...

### [小段標題 2]
...

---
```

**重點**：

- 開頭一定要有 `> 詳見 01X-... 第 X 章。[橋接邏輯]`
- 橋接邏輯講「這集跟既有原則的關係」，不是新觀點空降
- 結尾用 `---` 分隔，避免跟下一段混淆

### Step 5. 雙檢索更新（線性 + 話題）

`00-index.md` 兩處更新：

**更新 A：來源標記區**

```markdown
來源標記：[書] = ... ／[2025訪談] = ... ／**[2025訪談II] = ... ／[2026訪談] = ...**
```

**更新 B：每個既有主題下加 [YYYY訪談N] bullet**

讓搜「槓桿」這個主題的人會看到新訪談集的相關 bullet：

```markdown
### 槓桿、當責、股權 → `01c-leverage.md`
- 三種槓桿：勞力、資本、程式碼與媒體 [書]
- ...
- 純軟體不可投資 [2026訪談]   ← 新加
- 個人 App Store [2026訪談]   ← 新加
```

**更新 C：新增獨立主題段（給整集主檔露出）**

```markdown
### Vibe Coding 與軟體革命 → `01g-vibe-coding-2026訪談.md`
- [整集核心 bullet 列表]
```

**更新 D：話題對照表加問句**

```markdown
| Vibe Coding 是什麼 | `01g-vibe-coding-2026訪談.md` |
| 跟 AI 協作要注意什麼 | `01g-...` / `02a-clear-thinking` |
```

### Step 6. SKILL.md 加來源定義、話題路由

SKILL.md 兩處更新：

**A. 版本號**：v2.1 → v2.2

**B. 來源標記區**：

```markdown
`[2025訪談II]` = 2025 年 10 月 14 日，Naval Podcast，「In the Arena」...
```

**C. References 清單**：

```
├── 01g-vibe-coding-2026訪談.md       2026 訪談主檔
├── 01h-life-in-the-arena-2025訪談II.md  2025 訪談 II 主檔
```

**D. 話題路由**：把新主檔加進路由表

### Step 7. 中英文版（如有）一起做

如果你的設定檔有中英雙版（像 Naval 那樣），**兩邊一起更新**。不要中文做完才想到英文。

實踐：

1. 中文寫完 → 立刻翻譯英文版
2. 雙版的章節對接結構必須一致
3. 來源標記翻譯：`[2026訪談]` → `[2026 Interview]`、`[2025訪談II]` → `[2025 Interview II]`
4. 主檔檔名翻譯：`01h-Life-in-the-Arena-2025訪談II.md` → `01h-life-in-the-arena-2025-interview-ii.md`

### Step 8. 同步衍生系統（如果有）

如果這個設定檔被其他系統 import 了（例如村莊客座講師、其他工具箱），**一起同步**：

範例：Naval 設定檔同時存在三處：
- `_agent/skills/naval-ravikant/`（中文技能包）
- `_agent/skills/naval-ravikant-en/`（英文技能包）
- `01 數位分身小聚/01 數位分身小聚 主系統/05 NPC客座講師/GUEST-001-納瓦爾老師-naval_teacher/`（NPC 系統）
- `01 數位分身小聚/01 數位分身小聚 GitHub 版/...`（GitHub 部署版）

每次更新，**四處 references 都要同步**（用 `cp` 批次同步）。

如果衍生系統有自己的主檔（例如 NPC 主檔），也要更新（關鍵字、附件表、話題地圖）。

### Step 9. 備份 + 心智圖

**備份**：

```bash
TS=$(TZ=Asia/Taipei date +"%Y-%m-%d-%H%M")
cp -R "_agent/skills/[技能包名]/" "05 技能包備份/$TS [技能包名] vX.X/"
```

**心智圖**：`_agent/技能包心智圖.md` 找到對應條目，更新版本號跟更新摘要：

```markdown
- [技能包名]（`_agent/skills/...`）
  - vX.X（YYYY-MM-DD）：加入 [YYYY訪談N] 來源，新增 01X-...，01a/01c/02a 各加延伸段
  - 雙軌版；XX reference 檔
```

## 完整工作流程示意

```
[新訪談集進來]
      ↓
Step 1. 抓日期、定來源標記
      ↓

      ↓
Step 3. 建主檔（01X-...）
      ↓

      ↓

      ↓

      ↓
Step 8. 同步衍生系統（如有）
      ↓
Step 9. 備份 + 心智圖
      ↓
[完成]
```

## 常見錯誤

### 錯誤 1：直接建獨立檔案

新集進來建 `15-vibe-coding-2026.md` 就放進 references，沒對接到既有檔。
**結果**：讀者搜既有主題看不到新觀點。
**解法**：按 Step 4 對接到既有檔末尾。

### 錯誤 2：橋接邏輯是空降

「## 2026 訪談延伸」之後直接開始講新內容，沒解釋這集跟原本原則的關係。
**結果**：讀者讀到延伸段莫名其妙、不理解為什麼放這裡。
**解法**：每個延伸段開頭一定要有「這集跟既有原則 X 的關係是 Y」。

### 錯誤 3：中文做完忘了英文

中文版升 v2.1 之後過好幾週才想到英文版要更新。
**結果**：技術債累積，雙版不同步、學員看到不一致內容。
**解法**：Step 7 強制兩邊一起做。

### 錯誤 4：主檔太薄、沒有對接表

整集主檔只是 transcript 重貼，沒有開頭的「對接表」。
**結果**：讀者看不到這集跟既有原則的整體關係。
**解法**：Step 3 強制在主檔開頭加對接表（5-8 個對接點）。

---

## 提醒

這個 SOP 第一次跑會很慢（4-8 小時）。第二次以後會越來越快（1-3 小時），因為你已經熟悉這個人格的對接結構。

**效率關鍵：對接表寫好，後面就是體力活。** 不要急著動筆，先把 Step 2 的對接表想清楚。
