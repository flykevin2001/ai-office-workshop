# Gem 模式部署

> 把人格分身打包成可一鍵上架到 **Gemini Gem / ChatGPT GPTs / Claude Project** 的格式。同一套產物，三個平台通用。

---

## 完整版 vs 輕量版（雙形態維護）

人格分身有兩種部署形態，**同一個人格可以同時維護兩版**：

### 完整版（Agent 模式）
- **平台**：Claude Code、claude.ai Projects、本地 Agent SDK
- **字元限制**：無（SKILL.md 可像 Naval 13700 字元、references 可超過 10000）
- **使用方式**：放 `_agent/skills/<name>/`，技能包整套讀進
- **適合**：深度研究、長對話、完整脈絡

### 輕量版（GEM / GPTs 模式）
- **平台**：Gemini Gem、ChatGPT GPTs、Claude Project（對外公開）
- **字元限制**：主檔 ≤5000、分檔 ≤10000、總計 ≤45000
- **使用方式**：SKILL.md 貼系統指令、references 上傳知識檔
- **適合**：對外分享、手機隨時聊、上架公開

### 雙形態維護策略

**推薦策略 C：以輕量版為主，完整版同源**

```
_agent/skills/<name>/        ← 完整版（不限字元，給 Agent 跑）
  ├── SKILL.md
  └── references/

暫存/<name> 成品 放這裡/      ← 輕量版（壓進字元上限，給 GEM/GPTs）
  ├── SKILL.md                 (≤5000)
  ├── gem-meta.md
  └── references/              (每份 ≤10000)
```

兩個資料夾**共用同一個人格內容**，只是輕量版做了壓縮。

### SKILL.md 內容分配原則（輕量版）

主檔 5000 字元很緊。**檢索做好比塞細節重要**。

- ✅ **主檔留**：人格五段精簡 + 完整雙檢索
- 🚚 **拆到 references/**：所有細節範例、口頭禪清單、具體案例、慣用句、典故

理由：AI 讀主檔抓人格定位，遇到具體話題會去 references 找。檢索做得好，主檔可以瘦身。

### 後續優化方向（roadmap）

1. **壓縮工具**：把完整版 SKILL.md（10000+ 字元）自動萃取為輕量版（5000 字元）
2. **展開工具**：把輕量版補充細節到完整版（讓 Agent 跑更深）
3. **驗證工具**：用 100 個模擬問題測 GEM/GPTs vs Agent 的人格還原度差距
4. **自動同步**：完整版更新 → 自動 propagate 到輕量版
5. **多平台部署腳本**：一鍵把輕量版推上 Gemini Gem、GPTs、Claude Project

## 為什麼要 Gem 模式

技能包（SKILL.md + references/）原本是 Claude Code 內部使用。但使用者常希望把人格分身搬到 Gemini Gem / ChatGPT GPTs 等對外平台，跟更多人或自己手機上對話。

Gem 模式定義了一套「跨平台部署規格」，讓同一個分身可以同時上架三個平台，無需重做。

## 我們的規範（適配三平台）

### 主檔（SKILL.md）

- **字元上限：5000 字元**（中文約 2500-3300 字）含雙檢索
- **必含六段**：
  1. 我是誰（人格定位 + 一句話自介，~300 字元）
  2. 我從哪裡來（背景脈絡，~400 字元）
  3. 我怎麼說話（核心語氣 + 3-5 條具體規則，~500 字元）
  4. 我相信什麼（3-5 條核心信念，~500 字元）
  5. 我不做什麼（硬性邊界，3-5 條，~300 字元）
  6. **檢索頁（雙檢索）**：
     - **資料檢索**：references 檔案清單 + 一句說明（~500 字元）
     - **話題檢索**：按主題分類「你可以問我」→ 對應到哪份分檔（~2000 字元）
- **用途**：直接複製貼到 Gemini Gem / ChatGPT GPTs 的「系統指令」/「Instructions」欄
- **為什麼含檢索頁**：讓 AI 主動知道「使用者問 X，我該去哪份知識檔找答案」，避免亂引用

### 分檔（references/*.md）

- **每份字元上限：10000 字元**（中文約 5000-6500 字）
- **數量**：8-12 份（推薦 8-10 份）
- **命名**：`NN-主題.md`（如 `01-起點與創辦故事.md`）
- **內容規格**：
  - 按主題獨立拆分
  - 每份開頭有 1-2 句「這份是什麼」說明
  - 每份結尾「相關連結」指向其他份（wiki-link 風）
- **用途**：上傳成 Knowledge Files

### 元資料（gem-meta.md）

- **Greeting**：第一次見面的招呼語（30-100 字）
- **Conversation Starters**：3-4 個建議開場問題
- **Restricted Topics**：哪些不能說（隱私、未公開資訊等）
- **頭像建議**：選一張合適頭像（人物照、抽象象徵）
- **建議命名**：技能包名稱（如「V · 在地培力」）

## 三平台部署步驟

### Gemini Gem

1. 進入 Gemini → 「+ 建立 Gem」
2. 在「指令」欄貼 SKILL.md 內容
3. 在「知識」欄上傳 references/*.md（最多 10 個）
4. 填入 gem-meta.md 的 Greeting / Conversation Starters
5. 設定頭像、命名

### ChatGPT GPTs

1. 進入 ChatGPT → 「Explore GPTs」→「Create」
2. 在「Configure」→「Instructions」貼 SKILL.md 內容
3. 在「Knowledge」上傳 references/*.md（最多 20 個）
4. 填 Conversation Starters（4 個）+ Greeting
5. 設定頭像、命名

### Claude Project

1. 進入 Claude → 「+ New Project」
2. 在「Custom Instructions」貼 SKILL.md 內容（注意 ~5000 字元上限）
3. 在「Project Knowledge」上傳 references/*.md
4. 設定 Project 名稱

## 字元上限的緣由

| 上限 | 原因 |
|---|---|
| **主檔 3000 字元** | 經驗法則：超過 3000，AI 抓不住核心人格；少於 1500，會失去細膩度。3000 是甜蜜點。 |
| **分檔 10000 字元** | 單一主題深度展開的合理範圍；超過 10000 會稀釋焦點，建議拆兩份。 |
