# 學習地圖模板與問答流程

第一次使用技能包時建立，之後可更新。

存放位置：跟 SKILL.md 同層目錄，命名 `my-learning-map.md`。

---

## 8 問問答模板

AI 一次問一題，你回答完再問下一題。每題都有「為什麼問這個」幫你理解，避免亂答。

### [1/8] 主要工作 / 角色

**問**：你目前的主要工作或角色是什麼？用一句話描述。

**為什麼問**：學習地圖的根，是「你現在在做什麼」。所有後續判斷都圍繞這個。

**回答範例**：
- 「自由攝影師，接商業攝影案（人像、產品、活動）」
- 「中小企業行銷主管，做 B2B 軟體」
- 「想轉職的寫作者，目前還在累積作品」
- 「AI 教練，教中小企業老闆用 AI」
- 「全職媽媽，想學設計兼接案」

### [2/8] 工作流 / 工作節奏

**問**：你的工作分哪幾個階段或步驟？依時間順序列出。

**為什麼問**：老師教的東西要能接到工作流的某個環節，才有價值。沒有工作流，「學了能用嗎」就答不出來。

**回答範例**：
- 攝影師：接案 → 前置（場勘、設備）→ 拍攝 → 修圖 → 交片 → 後續（追修、發票）
- 行銷人：策略 → 內容企劃 → 投放 → 數據分析 → 優化迭代
- 寫作者：發想 → 大綱 → 寫稿 → 編輯 → 發布 → 反思
- 教練：開發 → 行銷 → 諮詢／教學 → 學員追蹤 → 復盤

### [3/8] 最卡住的環節

**問**：目前最常卡住的環節是哪個？描述一下為什麼卡。

**為什麼問**：「想學什麼」的真正答案藏在「卡在哪」裡。很多人說想學 X，其實是因為 Y 卡住，X 只是表面解法。

**回答範例**：
- 「修圖環節：每張要 5-10 分鐘，案子大時會塞車」
- 「內容企劃階段：我寫得出文案但抓不到差異化角度」
- 「發想階段：常常坐下來不知道要寫什麼」

### [4/8] 已掌握的技能

**問**：跟你工作相關的，你已經會的技能列 5 個（不要列「我還可以的」，列「我可以拿來收錢/交付」的）。

**為什麼問**：用來「找鄰居」。新東西要學，先看跟你已會的哪個最像，這樣評估老師講的內容是不是「你已知 80%」會更準。

**回答範例**（攝影師）：
1. 攝影基礎構圖、打光
2. Lightroom 批次處理
3. Photoshop 進階修圖（皮膚、色彩）
4. 客戶溝通、報價、合約
5. 設備管理與維護

### [5/8] 想精進的環節

**問**：哪一段最想補強？跟第 3 題的「卡住點」可能有關，也可能不一樣。

**為什麼問**：卡住的不一定是想精進的。例如修圖卡住，可能想精進的是「接案能力」（接更高單價案子，讓修圖時間相對划算）。

### [6/8] 學習偏好

**問**：你的學習偏好？看書／影片／一對一／實作／同儕（同學一起）。可以複選但講主要的。

**為什麼問**：老師再厲害，如果交付方式跟你的偏好嚴重不合，學習效率會打對折。例如你偏好實作，就少報純講課的線上課。

**回答範例**：
- 主要：影片 + 實作
- 補充：希望課程提供可下載的 preset / 範本

### [7/8] 時間預算

**問**：你的時間預算？每週能投入幾小時、能撐幾個月？

**為什麼問**：投資產出比的分母。再好的課程，超過你能投入的，就是不適合你。

**回答範例**：
- 每週可投入：5 小時
- 能撐：3 個月

### [8/8] 判斷準則

**問**：你的判斷準則？學完要能直接用，還是可以是長線投資？時間 cap 多少？

**為什麼問**：每個人對「值不值得」的標準不一樣。有人要立刻變現，有人能等三年。先講清楚，後續評估才不會打架。

**回答範例**：
- 時間 cap：超過 10 小時學不會的，先放著
- 直接套用要求：必須能直接接到現有 Lightroom 工作流，不接受全套換軟體
- 其他：學完三個月內要能拿來收錢

## 完整範例：攝影師

```markdown
# 我的學習地圖

更新日期：2026-05-04

## 1. 主要工作 / 角色

自由攝影師，接商業攝影案（人像、產品、活動為主）

## 2. 工作流 / 工作節奏

接案 → 前置（場勘、設備）→ 拍攝 → 修圖（Lightroom + Photoshop）→ 交片 → 後續（追加修改、申請發票）

## 3. 最卡住的環節

修圖環節：每張要 5-10 分鐘，案子大時會塞車。客戶要求越來越多細節，但報價沒辦法跟著漲那麼多。

## 4. 已掌握的技能

1. 攝影基礎構圖、打光
2. Lightroom 基本批次處理
3. Photoshop 進階修圖（皮膚、色彩）
4. 客戶溝通、報價、合約
5. 設備管理與維護

## 5. 想精進的環節

修圖效率：希望能用 AI 加速，但又怕品質掉。次要是：接案能力，接更高單價的案子。

## 6. 學習偏好

主要：影片 + 實作
補充：希望課程提供可下載的 preset / 範本

## 7. 時間預算

- 每週可投入：5 小時
- 能撐：3 個月

## 8. 判斷準則

- 時間 cap：超過 10 小時學不會的，先放著
- 直接套用要求：必須能直接接到現有 Lightroom 工作流，不接受全套換軟體
- 其他：學完三個月內要能拿來收錢
```

## 完整範例：行銷主管

```markdown
# 我的學習地圖

更新日期：2026-05-04

## 1. 主要工作 / 角色

中小企業行銷主管，B2B 軟體公司，帶 3 人團隊

## 2. 工作流 / 工作節奏

策略 → 內容企劃 → 投放（FB / Google / LinkedIn）→ 數據分析 → 優化迭代

## 3. 最卡住的環節

內容企劃：抓得到主題，但抓不到「差異化角度」。寫出來的東西跟競爭對手很像，沒有記憶點。

## 4. 已掌握的技能

1. 廣告投放（FB / Google）
2. SEO 基礎
3. GA4 數據分析
4. 團隊管理（3 人）
5. 內容寫作（產品文案、電子報）

## 5. 想精進的環節

差異化內容策略：能找到別人沒講的角度。次要是：用 AI 加速內容生產但不流於平庸。

## 6. 學習偏好

主要：一對一顧問 + 實作（同儕共學也 OK）
補充：不太喜歡純線上課，希望有真人互動

## 7. 時間預算

- 每週可投入：3 小時
- 能撐：6 個月

## 8. 判斷準則

- 時間 cap：超過 20 小時學不會的，看具體內容再說
- 直接套用要求：可以是長線投資（3-6 個月後看到效果 OK）
- 其他：團隊也能受惠的優先
```

## 更新規則

什麼時候要更新學習地圖？

- ✅ 你的主要工作改了（轉職、副業變主業）
- ✅ 工作流程做了大調整
- ✅ 之前的瓶頸解決了，有新瓶頸
- ✅ 已超過三個月沒更新
- ✅ 學完一堂大課之後，重新檢視地圖

更新方式：跟 AI 說「更新我的學習地圖」「我的工作改了」「重建學習地圖」。

AI 會：
1. 讀現有地圖
2. 問你哪幾項要改（不重問已答過的）
3. 只更新有變的項目
4. 在檔案頂部更新日期

## 進階：把學習地圖當定期回顧工具

學習地圖的副產品價值：每三個月跑一次更新，等於做一次工作流自我體檢。

可以問自己：
- 工作流變了沒？
- 卡住點移動了嗎？（卡住點變了 = 你進步了）
- 已掌握的技能列表多了哪些？
- 判斷準則變得更嚴還是更鬆？

把舊版本另存成 `my-learning-map-2026-Q1.md`、`my-learning-map-2026-Q2.md`，一年下來會看到自己的成長軌跡。
