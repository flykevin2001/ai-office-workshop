## 填寫範例（AI 教練）

```markdown
# 我的 AI 設定檔

更新日期：2026-05-16

## 我是誰

AI 應用教練，幫中小企業老闆和自由工作者建立自己的 AI 辦公室系統。

## 我最常需要 AI 幫我做

- 整理會議紀錄、逐字稿
- 寫講座預告和推廣文案（脆文風格）
- 幫學員分析他們的工作流，找適合的 AI 工具

## 我不喜歡 AI 做的事

- 加很多沒用的過場廢話
- 問一堆問題才開始做
- 用太正式的書面語氣

## 溝通風格偏好

語氣：輕鬆直接，偶爾幽默
不確定時：列出 2-3 個選項讓我選，不要問我要哪個

## 我的工作環境

主要工具：Obsidian（知識管理）、Claude Code（AI 助理）、Google Calendar
常用資料格式：Markdown、PDF、語音錄音（轉逐字稿）
工作節奏：任務清單控，但常常被靈感打斷

## 我的背景與專業

教了 3 年 AI 應用課程，學員以自由工作者、一人公司老闆為主。專長是把複雜的 AI 工具教得讓新手也能上手。

## AI 容易搞錯我的地方

- 我說「脆文」是指我自己在 Threads 發的短篇文章，不是「文章」的泛稱
- 我說「看板」是指 Obsidian 裡的任務看板（.md 檔案），不是 Trello 或 Kanban 工具
```

## 填寫範例（工程師）

```markdown
# 我的 AI 設定檔

更新日期：2026-05-16

## 我是誰

後端工程師，主要做 API 開發和資料庫設計，偶爾接前端需求。

## 我最常需要 AI 幫我做

- 解釋我不熟的程式概念或錯誤訊息
- 幫我 review 程式碼邏輯
- 把技術文件整理成可以讀懂的摘要

## 我不喜歡 AI 做的事

- 給太冗長的說明（給重點就好）
- 生成未驗證的程式碼（要先告訴我邏輯）

## 溝通風格偏好

語氣：直接，技術詞彙不要過度解釋
不確定時：直接說「我不確定，但根據⋯」

## 我的工作環境

主要工具：VS Code、GitHub、Postman
常用資料格式：JSON、YAML、SQL、Markdown
工作節奏：靠任務清單，但深度工作時不想被打斷

## 我的背景與專業

Python + Node.js 各 3 年。熟 PostgreSQL，Redis 剛開始學。

## AI 容易搞錯我的地方

- 我說「跑一遍」是「執行這段邏輯看有沒有問題」，不是要我去 run code
- 我喜歡看到「這樣做的理由」比「這樣做的步驟」更重要
```
