## 四個基本語法

### 1. 章節標題

```
# 大標題
## 小標題
### 更小的標題
```

用途：區分內容邊界。AI 靠這些標題判斷「這一段在講什麼」。一個標題段落就是一張卡片。

老師的核心觀念：「標題不只是讓文章好看，它是你在這篇筆記裡畫出的最小內容邊界。」

應用場景：
- 資料夾分大脈絡（教學脈絡 / 外部參考 / 學生情況）
- 文章內用標題分小段落（摘要 / 概念 / 執行方法）
- AI 就能知道每一段從哪裡到哪裡

### 2. 標籤

```
#學習 #AI #財富
```

注意：井字號後面直接接文字，沒有空格。跟 IG 的 # 一模一樣。

用途：快速搜尋。點一個標籤，所有帶這個標籤的筆記都會出來。

不知道怎麼設標籤 → 叫 AI 幫忙：「幫我設幾個關鍵字標籤」

### 3. 卡片連結

```
[[筆記名稱]]
```

用途：標註筆記之間的關係。讓 AI 知道 A → B → C 的因果。

老師的 CP 值最高做法：至少標一條最簡單的連結，講一下上一篇是什麼就好。

例如在 B 執行方案裡寫：「這是 [[A 會議紀錄]] 的延伸」。AI 就知道先有 A 才有 B。

### 4. 條列

```
- 第一項
- 第二項
- 第三項
```

減號 + 空格 就是條列。最基本的整理方式。

---

## 小提醒：標題 vs 標籤

標題：`# 空格 文字` → 會變成大字的段落標題
標籤：`#文字`（沒有空格）→ 會變成可點擊的標籤

搞錯空格位置會出問題，記住這個差異就好。

## 進階概念

### 內容邊界

不用把每個概念都拆成一個獨立檔案。最簡單的做法就是用 # 標題切出內容邊界。

### 受控詞彙

標籤越來越多的時候需要統一管理。同一個概念只用一個詞。
例如：「初階」「入門」「基礎」「簡單」統一成 #基礎

### 後設資料

描述文件本身的資料。YAML 格式，放在筆記最上方。老師建議晚一點再整理。

### 四維度標籤管理法

D1 對象：這篇跟「誰」有關
D2 內容：講「什麼主題」
D3 情境：在「什麼情境」下用
D4 狀態：現在是「什麼狀態」
