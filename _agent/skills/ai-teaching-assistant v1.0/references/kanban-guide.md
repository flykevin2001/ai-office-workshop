# 看板管理完整操作指南

## 使用環境

Claude Cowork + Obsidian（Codex 也可以操作，但 Cowork 安全機制較完善）

## 看板格式

看板是一份普通的 .md 檔案，搭配 Obsidian Kanban 插件使用。

檔案開頭需要這段 YAML：

```
---
kanban-plugin: board
---
```

每一列代表一天，用二級標題表示：

```
## 04-05 六
```

列底下放卡片，用 Markdown 勾選格式：

```
## 04-05 六

- [ ] 09:00 開會
- [ ] 整理筆記
- [x] 回覆信件
```

卡片格式規則：
- 有時間：`- [ ] HH:MM 事項名稱`
- 沒時間：`- [ ] 事項名稱`
- 有對應筆記：`- [ ] [[筆記連結|HH:MM 事項名稱]]`
- 完成打勾：`- [x] 事項名稱`
- 不確定的事也要先開卡，標上 #尚未確認 或 #規劃中

Kanban 插件來源：https://github.com/mgmeyers/obsidian-kanban
沒裝插件也能用，只是看起來會是普通的 Markdown 清單，功能不受影響。

## 三個核心功能

### 功能一：開卡

觸發：「開卡」「加到看板」「記一下」「幫我排進去」

流程：
1. 讀取使用者的看板檔案
2. 沒有看板檔 → 問使用者要不要建一份
3. 判斷這張卡片要放在哪一天的列底下
   - 使用者有說日期 → 放到對應日期
   - 沒說日期 → 問一句「這件事預計什麼時候做？」
4. 寫入看板

### 功能二：查看板

觸發：「今天要做什麼」「看板狀態」「最近有什麼」

流程：
1. 讀取看板檔案
2. 找到今天（或使用者指定的日期範圍）的列
3. 回報：哪些還沒做、哪些已完成
4. 如果有過期未完成的卡片，主動提醒

### 功能三：整理看板

觸發：「整理看板」「看板太長了」

流程：
1. 掃描看板，找出超過 30 天的列
2. 該列全部完成 → 建議搬到備存檔
3. 該列還有未完成 → 問使用者怎麼處理（忘了？要移到新日期？要取消？）
4. 使用者確認後執行
5. 備存位置：使用者自己指定，或預設放在看板同資料夾的「看板備存/」

## 操作原則

- 所有動作先建議、等確認、再執行
- 不確定的事也開卡，標 #尚未確認
- 卡片只寫「時間 + 事項名稱」，保持精簡
- 詳細資訊寫在對應的筆記檔裡，卡片只放連結
- 不刪除卡片，舊的備存起來
- 不動使用者的其他檔案，只處理看板

## 老師的實際用法

老師目前有四個看板：
- 工作看板：零碎工作、個人日常任務
- 發文看板：社群貼文（脆文、方格子等）
- 邀約看板：跟人有約的事（會面、約聊、合作邀約）
- 課程看板：講座、課程場次

老師怎麼跟 AI 配合：
- 「幫我查一下我今天要做什麼？」→ AI 直接查看板，報告未完成項目
- 「我明天好像比較空，這個月有什麼東西要再整理一下？」→ AI 掃描看板回報
