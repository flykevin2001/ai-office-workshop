# 為什麼需要這套系統

這套系統處理的是四個核心問題：

---

## PDF 轉純文字：資料清理

PDF 是印刷格式，是設計給人看的，不是設計給 AI 看的。

我們看到的漂亮排版，實際上都是程式碼。每一個字多大、多黑、什麼字型、放哪個位置，座標都要寫進去。一條分隔線，我們看到很清楚就是一條線，但對 PDF 本身的結構來說，它要描述「有一條線，它的原點是哪裡，從哪裡到哪裡」。光是這條線就有大量的座標位置筆劃資訊。

表格更麻煩。我們看到的是很清楚的一行字，但所有的表格框線、位置、大小全部都要寫清楚，所以就會有大量的亂碼。對 AI 來說，它還要再去查座標，同樣座標放在一起的，它才認定是同一列。如果不小心多打了一個空格，讓座標對不起來，AI 可能就覺得不是同一列，甚至判斷成下一列。

簡單說就是「資料清理」：去除掉多餘的排版規格與亂碼，只保留純文字，內容才能更好的被 AI 處理、被搜尋引擎找到、被其他工具串接。

概念就是：盒子越簡單，干擾就越少。

## 翻譯為什麼要獨立一個階段：上下文爆炸

翻譯長文件，建議不要在同一個視窗一路翻到底。

翻譯第一章的時候，AI 要讀整份原文加第一章的中文。翻譯第二章的時候，AI 要把全部原文再讀一遍，再讀第一章的中文，然後才翻第二章。到第三章可能就爆炸了。

因為 AI 沒有辦法一口氣輸出，它要逐段產出。但每一次請它逐段輸出，它就要重新讀前面的內容，這就造成上下文爆炸。累積到後面，速度越來越慢，甚至直接觸發長度上限，翻譯中斷。

而且沒有統一的翻譯規範，第一章翻一次，第二章重新開始翻，用詞跟結構可能完全不一樣。同一個術語前面翻「新創公司」，後面翻成「創業企業」，語氣從口語化變成文謅謅，段落結構開始跑掉。

所以翻譯需要獨立成一個階段，搭配字典檔和翻譯規範，確保每一章分開翻、用語統一、格式一致。

---

## 卡片化：讓知識能被重複使用

AI 的注意力機制有一個重要特性：丟越多東西給它，它就會越混亂、越渙散，跟人類一樣。你沒有跟它講重點，丟一堆東西給它，它就搞混了、頭昏了。資料量跟專注力是成平方比的，資料量一多，注意力跟準確度就會雪崩式下滑。

所以與其把一整本書塞在一個檔案裡讓 AI 去讀，不如拆成一章一張卡片。每張卡片是獨立的知識單位，AI 每次只需要專注處理一小塊，準確度自然就上來了。而且拆開之後，每張卡片可以單獨搜尋、連結、引用。

這其實就是最簡單的卡片盒筆記法應用，把知識拆解成單張單張的卡片，需要的時候再依照需求重新組合。最簡單的概念就是積木，積木平常是散在那邊，但需要的時候就可以立刻組合出要用的模式。

AI 可以快速調用不同段落，幫你組成新的內容。寫企劃書的時候會非常實用，可以把好幾篇文章裡面的不同段落清楚調用，組成一篇新的文章。

所以卡片化不是為了好看，是為了讓知識能真正被重複使用。

## Markdown 格式：劃分內容邊界

沒有標題的資料，AI 不知道第一段在哪裡結束、第二段從哪裡開始，整個內容就糊成一團。它只能猜，猜錯就會把不同段落的內容混在一起。

有了 Markdown 標題之後，AI 就很清楚，這個標題到下一個標題之間，就是一個完整的段落。碰到下一個同級標題，就知道那是新的段落了。不會搞混哪一段是哪一個。

Markdown 是目前最通用的文字格式語法，人看得懂、AI 也看得懂。你每次跟 ChatGPT、Claude、Gemini 對話，它們輸出的全部都是 Markdown 語法。連 Apple 最近的 Mac 大更新，備忘錄都直接支援 Markdown 了。

而且格式越原始，干擾越少。最好的格式是 TXT 純文字檔，或者 MD 檔。MD 檔不管丟到哪裡都長一樣，丟到支援它的軟體裡，就會自動幫你排版。所以這套系統輸出的格式以 Markdown 為主，確保內容邊界清楚、跨平台通用、AI 和人類都好閱讀。
